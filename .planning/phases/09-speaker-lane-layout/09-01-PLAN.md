  - `setSpeakerLane(speakerId: string, position: number)` — pushes undo, sets `speakerLanes[speakerId].verticalPosition = position`

  - `setMaxVisibleRows(n: number)` — pushes undo, clamp 1-10

**SubtitleOverlay.tsx** — Replace the `assignSlots()` greedy algorithm with fixed lane-based positioning:

1. Remove `assignSlots()` function and `OVERLAP_OFFSET_PCT` constant.
2. Add a `getLanePosition()` function:
   ```typescript
   function getLanePosition(
     phrase: CompositionPhrase,
     speakerLanes: Record<string, { verticalPosition: number }>,
     defaultPosition: number,
     overlapGap: number,
     sameSlotOffset: number,
   ): number {
     // Precedence: phrase styleOverride > speaker lane > global default
     if (phrase.styleOverride?.verticalPosition != null) {
       return phrase.styleOverride.verticalPosition - sameSlotOffset * overlapGap
     }
     const speaker = phrase.dominantSpeaker
     const lane = speaker ? speakerLanes[speaker] : undefined
     const basePosition = lane?.verticalPosition ?? defaultPosition
     return basePosition - sameSlotOffset * overlapGap
   }
   ```
3. In the rendering logic, where `slotMap` was used to compute `top`, replace with:
   - Keep `latestBySpeaker` map for same-speaker replacement logic (unchanged)
   - For each visible phrase, compute its `sameSlotOffset` (0 for the latest phrase of its speaker, 1+ for lingering older phrases of the same speaker)
   - Call `getLanePosition()` to get the final `top` value
4. Implement `maxVisibleRows` cap: after building the visible phrases list, if count exceeds `maxVisibleRows` (default 4), sort by earliest word start descending (newest first) and take only the first N.
5. Use `overlapGap ?? 8` as fallback for backward compatibility.
