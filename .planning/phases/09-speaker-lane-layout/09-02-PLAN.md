2. Set `pragma('journal_mode = WAL')`
3. Create table:
   ```sql
   CREATE TABLE IF NOT EXISTS lane_presets (
     id TEXT PRIMARY KEY,
     name TEXT NOT NULL,
     layout TEXT NOT NULL,
     created_at INTEGER NOT NULL,
     updated_at INTEGER NOT NULL
   )
   ```
   The `layout` column stores JSON string of `{ speakerLanes: Record<string, { verticalPosition: number }>, overlapGap: number, maxVisibleRows: number }`
4. Decorate `fastify.lanePresetsDb` (NOT `fastify.db` — that's taken by animationPresets)
5. Register onClose hook to close the DB
6. Export the plugin function wrapped with `fp()` for encapsulation breaking
7. No built-in presets — user-created only

**routes/lanePresets.ts** — Create CRUD routes mirroring `presets.ts`:

- `GET /api/lane-presets` — Returns all presets sorted by `created_at ASC`. Each preset has `id`, `name`, `layout` (parsed from JSON), `createdAt`, `updatedAt`.
- `POST /api/lane-presets` — Body: `{ name: string, layout: { speakerLanes, overlapGap, maxVisibleRows } }`. Generates UUID id, stores layout as JSON string. Returns 201 with created preset.
- `PUT /api/lane-presets/:id` — Body: `{ name?, layout? }`. Merges provided fields. If layout provided, replaces entire layout JSON. Updates `updated_at`. Returns 200 with updated preset.
- `DELETE /api/lane-presets/:id` — Deletes preset. Returns 204.

Use `crypto.randomUUID()` for IDs (same pattern as animation presets).
