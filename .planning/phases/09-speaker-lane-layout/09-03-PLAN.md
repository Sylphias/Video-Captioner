**LaneDragOverlay.tsx** — Absolute overlay on the Remotion Player:

Props: `speakerLanes`, `speakerNames`, `containerRef` (ref to player wrapper for height measurement), `currentFrame` (from Remotion player state), `phrases` (CompositionPhrase[] from store).

**Per-speaker drag handles:**
For each speaker in `speakerLanes`, render a horizontal dashed line:
- `position: absolute; left: 0; right: 0; top: ${position}%`
- `border-top: 2px dashed ${speakerColor}` using `getSpeakerColor()` (copy from SpeakerStylePanel.tsx or extract to shared util in `packages/frontend/src/utils/speakerColors.ts`)
- Speaker name label on the left end of the line (small, colored, semi-transparent background)
- `cursor: ns-resize` on the line for drag affordance
- `pointer-events: auto` on drag handles, `pointer-events: none` on the overlay container itself

**Per-phrase drag handles (Blocker fix):**
In addition to per-speaker handles, render a per-phrase drag handle for each phrase currently visible at `currentFrame`:
- Determine visible phrases by checking which phrases' time range includes `currentFrame` (convert frame to seconds using fps)
- For each visible phrase, render a shorter dashed line (right-aligned, ~40% width) at the phrase's effective vertical position
- Use a distinct style: `border-top: 2px dotted ${speakerColor}; opacity: 0.7` with a small label showing phrase index/text snippet
- Drag interaction on per-phrase handle calls a phrase style override setter (e.g., `updatePhraseStyleOverride(phraseIndex, { verticalPosition: newPos })`) — use the existing phrase styleOverride mechanism from the store
- Per-phrase handles only appear when a phrase is currently visible; they overlay slightly below/offset from the speaker handle to avoid visual collision
- If the phrase already has a `styleOverride.verticalPosition`, show the handle at that position; otherwise show it at the speaker lane position (indicating it would create an override on drag)

**Shared drag interaction** (follows existing `mousedown → document.mousemove → document.mouseup` pattern):
1. `onMouseDown` on a handle: push undo snapshot via `useSubtitleStore.getState()`, record `startY` and `startPosition`
2. `document.addEventListener('mousemove', onMove)`: compute `deltaPercent = ((moveE.clientY - startY) / containerHeight) * 100`, clamp new position to 5-95%
3. For speaker handles: call `setSpeakerLane(speakerId, newPosition)`
4. For phrase handles: call the phrase styleOverride setter with `{ verticalPosition: newPosition }`
5. `document.addEventListener('mouseup', onUp)`: remove listeners

**LaneControlsPanel.tsx** — Left-side panel beside the preview:

Layout: Narrow panel (~180px width) with:
1. **Per-speaker rows**: For each speaker, show:
   - Speaker color dot + name (from `speakerNames` store field)
   - Numeric input for vertical position (0-100, step 1, suffix "%")
   - Proximity warning icon (amber ⚠) if within 10% of another speaker
2. **Global controls section**:
   - "Overlap Gap" numeric input (1-30%, default 8)
   - "Max Rows" numeric input (1-10, default 4)
3. **Preset section**:
   - Dropdown select for saved lane presets (from `useLanePresets` hook)
   - "Save" button (saves current layout as new or updates existing)
   - "Delete" button (removes selected preset)
   - Save prompts for name via `window.prompt()` (simple, matches existing pattern)

Numeric inputs call `setSpeakerLane`, `setOverlapGap`, `setMaxVisibleRows` from store.

When loading a preset: call `loadLaneLayout(preset.layout)` on the store. Map positions to current speakers by sorting preset positions descending and current speaker IDs by their current position descending (or alphabetically if no positions yet).

Proximity warning: `hasProximityWarning()` checks all pairwise distances. If any pair < 10%, show amber warning icon next to both affected speakers. Use `color: var(--color-warning, #d4a047)`.

CSS: Dark theme consistent with existing panels. Use tokens.css variables. Compact layout since the panel is narrow.
