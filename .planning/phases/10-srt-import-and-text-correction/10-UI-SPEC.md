## Color

All values from `tokens.css`. Source: `packages/frontend/src/styles/tokens.css`.

| Role | Value | CSS Variable | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `#1a1a1a` | `--color-bg-base` | Page background, outer container |
| Secondary (30%) | `#242424` / `#2e2e2e` | `--color-bg-surface` / `--color-bg-elevated` | SrtDiffView panel background (`--color-bg-surface`), individual diff row background on hover (`--color-bg-elevated`) |
| Accent (10%) | `#4caf72` | `--color-accent-green` | Accept button background, "Import SRT" button hover border, active-phrase line-number |
| Destructive | `#e05454` | `--color-error` | Reject button hover state only |

**Accent reserved for:**
1. "Accept" button background (filled style)
2. "Import SRT" button border on hover (matches existing `.text-editor__add-line-btn:hover` pattern)
3. Diff-added text highlight background (using `--color-accent-green-muted` at reduced opacity, consistent with focus style in TextEditor)

**Diff highlight colors (Claude's Discretion ŌĆö per CONTEXT.md):**
- Added text (SRT words not in Whisper): background `--color-accent-green-muted` (`#1e3d2a`), text `--color-accent-green` (`#4caf72`)
- Removed text (Whisper words not in SRT): text `--color-text-disabled` (`#585858`), `text-decoration: line-through`, no background fill
- Unchanged text: `--color-text-primary` (`#e8e8e8`), no decoration

Rationale: Added = positive change (green = approve), Removed = obsoleted text (muted/struck through without alarming red). This keeps the destructive red (`--color-error`) reserved exclusively for the Reject button, preventing color-meaning collision.

## Interaction Contract

### Import flow (D-01, D-03)

1. User clicks "Import SRT" ŌåÆ hidden `<input type="file" accept=".srt">` opens OS file picker
2. User selects `.srt` file ŌåÆ `File.text()` reads content ŌåÆ `parseSrt()` + `alignSrtToWhisper()` run synchronously in browser
3. On success: `useSrtImport` status becomes `'parsed'`, `SrtDiffView` appears below import button
4. On failure: error state replaces diff panel; import button remains active for retry

### Per-phrase accept (D-08, D-09)

1. User clicks "Accept" on a row
2. `applySrtPhrase(phraseIndex, replacementWords)` called ŌĆö pushes undo snapshot first (D-09)
3. Row is removed from diff view immediately
4. Alignment re-runs against updated `session.phrases` to fix phrase index drift (RESEARCH.md Pitfall 2 ŌĆö Option A)
5. Remaining rows re-render with corrected indices

### Per-phrase reject

1. User clicks "Reject" on a row
2. Row removed from diff view; store untouched
3. No undo step created (nothing was changed)

### Diff panel dismiss

1. User clicks ├Ś on panel header
2. `useSrtImport.reset()` called ŌĆö status returns to `'idle'`, diff panel unmounts
3. Any un-accepted SRT phrases are discarded; accepted ones already applied via undo history

### Re-import (D-12)

1. User clicks "Import SRT" while diff panel is open
2. `useSrtImport.reset()` called first ŌĆö pending diff discarded
3. New file selected ŌåÆ alignment runs against `store.original` words (not current session) per RESEARCH.md Pitfall 6

### Keyboard

No new keyboard shortcuts defined for this phase. Undo (Cmd+Z / Ctrl+Z) applies to accepted phrases via existing undo system (D-09).
