Create `packages/backend/src/routes/transcribe.ts` — a Fastify route plugin (wrapped in `fp()`, matching the existing pattern in upload.ts and jobs.ts) that triggers transcription on a job.

**Route: POST /api/jobs/:jobId/transcribe**

Behavior:
1. Look up job by jobId in `fastify.jobs`
2. Validate job exists and is in `'ready'` or `'transcribed'` status (allow re-transcription)
3. Return 404 if job not found, 409 if job is in a non-ready state (uploading, normalizing, transcribing)
4. Set job status to `'transcribing'`, progress to 0
5. Reply 202 Accepted immediately: `{ jobId, status: 'transcribing' }`
6. Fire-and-forget: call `runTranscriptionPipeline()` (async, not awaited)

**`runTranscriptionPipeline` function** (in same file, not exported):

```typescript
async function runTranscriptionPipeline(
  fastify: FastifyInstance,
  jobId: string,
  jobDir: string,
): Promise<void> {
  try {
    const normalizedPath = path.join(jobDir, 'normalized.mp4')
    const transcriptPath = path.join(jobDir, 'transcript.json')

    const { promise, process: proc } = runTranscription(
      normalizedPath,
      transcriptPath,
      (percent) => {
        updateJob(fastify.jobs, jobId, { progress: percent })
      },
    )

    // Store process handle on job for zombie prevention
    // Use a WeakMap or side Map to avoid serializing ChildProcess into job SSE
    transcriptionProcesses.set(jobId, proc)

    await promise

    transcriptionProcesses.delete(jobId)

    updateJob(fastify.jobs, jobId, {
      status: 'transcribed',
      progress: 100,
      transcriptPath,
    })
  } catch (err) {
    transcriptionProcesses.delete(jobId)
    const errorMessage = err instanceof Error ? err.message : String(err)
    updateJob(fastify.jobs, jobId, {
      status: 'failed',
      error: errorMessage,
    })
    fastify.log.error({ jobId, err }, 'Transcription pipeline failed')
  }
}
```

**Subprocess tracking for zombie prevention:**

Declare a module-level `Map<string, ChildProcess>` called `transcriptionProcesses` to track active subprocess handles by jobId. This map is NOT part of the job store (ChildProcess is not serializable). It enables the SSE cleanup handler (in jobs.ts, Plan 02-03 Task 2) to kill orphaned subprocesses.

Export a function `killTranscription(jobId: string): void` that checks the map, calls `proc.kill()`, and removes the entry. This will be called from jobs.ts if the SSE client disconnects during transcription.

```typescript
// Module-level subprocess tracking
const transcriptionProcesses = new Map<string, ChildProcess>()

export function killTranscription(jobId: string): void {
  const proc = transcriptionProcesses.get(jobId)
  if (proc) {
    proc.kill()
    transcriptionProcesses.delete(jobId)
  }
}
```

   ```typescript

   // Note: 'ready' is NO LONGER terminal — user triggers transcription next
   ```

   IMPORTANT: Remove `'ready'` from the terminal state check. The SSE stream should stay open through `ready` so the frontend can monitor the job continuously through upload -> normalize -> ready -> transcribe -> transcribed.

2. **Strip `transcriptPath` from SSE broadcast** — Before `JSON.stringify(job)` in the SSE write, create a copy of the job object without `transcriptPath` (it's a server filesystem path that must not be exposed to the client):
   ```typescript
   const { transcriptPath, thumbnailPath, ...safeJob } = job
   reply.raw.write(`data: ${JSON.stringify(safeJob)}\n\n`)
   ```
   Also strip `thumbnailPath` — it was already being sent but is an internal path. The client accesses thumbnails via `/api/jobs/:jobId/thumbnail`.

3. **Zombie subprocess cleanup on SSE disconnect** — In the existing `req.raw.on('close', ...)` handler, add a call to kill any active transcription subprocess:
   ```typescript
   req.raw.on('close', () => {
     clearInterval(interval)
     // Kill transcription subprocess if still running (prevents zombie — Pitfall 4)
     killTranscription(jobId)
   })
   ```
   Import `killTranscription` from `../routes/transcribe.ts`.

4. **Add GET /api/jobs/:jobId/transcript endpoint** — Serves the transcript JSON content (not the file path). This is how the frontend retrieves the completed transcript:
   ```typescript
   fastify.get('/api/jobs/:jobId/transcript', async (req, reply) => {
     const { jobId } = req.params as { jobId: string }
     const job = fastify.jobs.get(jobId)

     if (!job) {
       return reply.code(404).send({ error: 'Job not found' })
     }
     if (!job.transcriptPath) {
       return reply.code(404).send({ error: 'Transcript not yet available' })
     }

     // Read and serve the transcript file content
     const { readFile } = await import('node:fs/promises')
     try {
       const content = await readFile(job.transcriptPath, 'utf-8')
       reply.header('Content-Type', 'application/json')
       return reply.send(content)
     } catch {
       return reply.code(404).send({ error: 'Transcript file not found on disk' })
     }
   })
   ```

NOTE on import circularity: `jobs.ts` imports `killTranscription` from `transcribe.ts`. `transcribe.ts` does NOT import from `jobs.ts`. The dependency is one-directional, so no circular import issue.
