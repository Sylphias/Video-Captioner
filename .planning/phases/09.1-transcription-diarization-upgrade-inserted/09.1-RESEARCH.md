## Summary

This phase upgrades two Python scripts (`transcribe.py`, `diarize.py`) from CPU-only Apple Silicon models to NVIDIA CUDA on Windows. The primary path is Parakeet TDT (NeMo) for transcription + NeMo MSDD for diarization. The fallback is faster-whisper large-v3 CUDA + pyannote CUDA.

**Critical finding:** NeMo officially does not support Windows via pip. The PyPI package explicitly states "No support yet" for Windows AMD64. The `triton` dependency (required by nemo_toolkit) has no official Windows wheel — a community fork (`triton-windows`) is the only workaround and is not guaranteed reliable. This means the Parakeet/NeMo path requires a validation spike to confirm installability before committing. The fallback (faster-whisper CUDA + pyannote.audio) works natively on Windows with well-documented pip install steps.

The integration surface is small: only the two Python scripts and two `.ts` service wrappers need updating. The Node.js subprocess spawn pattern, JSON-line progress protocol, and transcript.json format all stay unchanged.

**Primary recommendation:** Structure the phase as a spike-first approach. Task 1: validate Parakeet TDT installs on Windows and produces acceptable word timestamps. Task 2: validate NeMo MSDD installs and runs. If either spike fails, fall through to faster-whisper CUDA + pyannote CUDA, which is well-proven on Windows.

### Recommended Project Structure

### Pattern 1: Python subprocess per job (unchanged)
**What:** Each transcription/diarization job spawns a fresh Python subprocess. Node.js spawns `.venv\Scripts\python.exe` with the script path + args.
**When to use:** Always — this pattern is proven and preserved.
**Example (transcription.ts change — D-08):**
```typescript
// Before (Mac):
const PYTHON = path.join(REPO_ROOT, '.venv/bin/python')

// After (Windows):
const PYTHON = path.join(REPO_ROOT, '.venv', 'Scripts', 'python.exe')
```

### Pattern 2: JSON-line progress protocol (unchanged)
**What:** Python script emits `{"type": "progress", "percent": N}` to stdout; `{"type": "done", ...}` at completion; `{"type": "error", ...}` + exit 1 on failure.
**When to use:** Always — frontend SSE pipeline depends on this; must not change.

### Pattern 3: Parakeet TDT transcription
**What:** Load model once per process, transcribe audio file, emit word timestamps.
**Example:**
```python
# Source: https://huggingface.co/nvidia/parakeet-tdt-0.6b-v2
import nemo.collections.asr as nemo_asr

asr_model = nemo_asr.models.ASRModel.from_pretrained(
    model_name="nvidia/parakeet-tdt-0.6b-v2"
)
# Move to CUDA — model loads on CPU by default
asr_model = asr_model.to('cuda')
asr_model.eval()

output = asr_model.transcribe(['audio.wav'], timestamps=True)

# Word-level timestamps
word_timestamps = output[0].timestamp['word']
# Each entry: {'start': float, 'end': float, 'word': str}

# NOTE: Parakeet does NOT output per-word confidence scores natively.
# Use 1.0 as placeholder or derive from segment confidence if available.
for stamp in word_timestamps:
    words.append({
        "word": stamp['word'],
        "start": round(stamp['start'], 3),
        "end": round(stamp['end'], 3),
        "confidence": 1.0,  # Parakeet does not expose per-word probability
    })
```

### Pattern 4: faster-whisper CUDA (fallback path)
**What:** Upgrade existing transcribe.py to use CUDA device with float16.
**Example:**
```python
# Source: https://github.com/SYSTRAN/faster-whisper
from faster_whisper import WhisperModel

# Change from CPU to CUDA — all other logic stays identical
model = WhisperModel("large-v3", device="cuda", compute_type="float16")
# Word timestamps, VAD, segment iteration — unchanged from current code
```

### Pattern 5: pyannote diarization CUDA (fallback path)
**What:** Upgrade existing diarize.py to use CUDA device instead of CPU.
**Example:**
```python
# Change ONLY the device — all pipeline logic stays identical
pipeline.to(torch.device("cuda"))  # was "cpu"
# ffmpeg call: use "ffmpeg" not "/opt/homebrew/bin/ffmpeg"
```

### Pattern 6: Parakeet expects WAV input
**What:** Parakeet TDT accepts `.wav` and `.flac` at 16kHz mono. MP4 is not accepted.
**Action required:** transcribe.py must extract WAV from the MP4 via ffmpeg before calling Parakeet. Same pattern already used in diarize.py.
```python
# Pre-extract audio from mp4 to wav for Parakeet
subprocess.run(
    ["ffmpeg", "-y", "-i", audio_path, "-vn", "-acodec", "pcm_s16le",
     "-ar", "16000", "-ac", "1", wav_path],
    capture_output=True, check=True
)
output = asr_model.transcribe([wav_path], timestamps=True)
```

### Anti-Patterns to Avoid
- **Installing NeMo without triton-windows first:** On Windows, `pip install nemo_toolkit[asr]` will fail because `triton` (Linux-only official wheel) is a dependency. Must install `triton-windows` first.
- **Installing PyTorch after other packages:** WhisperX and pyannote can pull in CPU-only torch. Always install PyTorch CUDA wheel first.
- **Using device="cpu" as fallback:** D-09 prohibits CPU fallback — CUDA is hardcoded by design.
- **Hardcoding `/opt/homebrew/bin/ffmpeg`:** `diarize.py` line 62 has this hardcoded — must change to `"ffmpeg"` (on PATH, D-07).
- **Using `.venv/bin/python` path in TypeScript:** Both `transcription.ts` and `diarization.ts` use the Unix path — must change to `.venv/Scripts/python.exe` (D-08).
- **Assuming Parakeet outputs per-word confidence:** The model does not expose per-word probability. Must use a placeholder (e.g., 1.0) or derive from segment-level if available.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Word alignment | Custom forced aligner | faster-whisper (built-in CTC) or WhisperX (wav2vec2 alignment) | Alignment is a complex acoustic problem; CTC-based aligners handle silence, noise, accents |
| Speaker segmentation | Threshold-based energy detector | pyannote or NeMo MSDD | Speaker boundary detection requires embedding + clustering; hand-rolled solutions have poor accuracy on overlapping speech |
| CUDA memory management | Custom batch size calculator | NeMo's built-in batching / faster-whisper chunk handling | Both handle OOM gracefully with their own chunk strategies |
| WAV extraction from MP4 | Python audio library | ffmpeg subprocess (already in diarize.py) | ffmpeg handles all codec edge cases |
| Progress % during NeMo transcription | Custom segment counter | NeMo hook callbacks (if available) or estimate from audio duration | NeMo may not expose segment-level progress; use time-based estimation |

**Key insight:** The hardest parts of this domain (forced alignment, speaker clustering, neural diarization) are fully solved by existing libraries. The integration work is mostly path updates and output format mapping.

## Critical Blocker: NeMo on Windows

### What's blocked
`nemo-toolkit` 2.7.2 (released 2026-03-25) officially states "No support yet" for Windows AMD64. The `triton` package (Linux-only) is a required dependency. Installing `nemo_toolkit[asr]` on Windows fails with "Could not find a version that satisfies the requirement triton."

### Workaround
`triton-windows` 3.6.0 (released 2026-03-10, moved to official `triton-lang/triton-windows`) provides Windows wheels for Python 3.10–3.14. Install it before nemo_toolkit:
```bash
pip install triton-windows
pip install "nemo_toolkit[asr]"
```

### Spike required
This workaround is community-maintained and not officially endorsed by NVIDIA. The spike must confirm:
1. `triton-windows` + `nemo_toolkit[asr]` installs without errors on Python 3.12 / Windows 11 / CUDA 13.1
2. `ASRModel.from_pretrained("nvidia/parakeet-tdt-0.6b-v2")` loads and moves to CUDA
3. `asr_model.transcribe(['test.wav'], timestamps=True)` returns `output[0].timestamp['word']` with valid start/end times
4. Word boundaries are at least as precise as current faster-whisper large-v3

### If spike fails → fallback
faster-whisper on CUDA Windows is straightforward and already in production (same library, just `device="cuda"`). pyannote.audio 4.0.4 supports Windows CUDA. Both are well-documented working paths.

## Common Pitfalls

### Pitfall 1: PyTorch CUDA wheels overwritten by pyannote/whisper
**What goes wrong:** pyannote.audio or whisperx installs CPU-only `torch` as a dependency, replacing the GPU-enabled torch.
**Why it happens:** pip resolves `torch` without CUDA extras unless you force it.
**How to avoid:** Always install PyTorch CUDA wheel first, then install other packages. Use `pip install --no-deps pyannote.audio` if it tries to reinstall torch.
**Warning signs:** `torch.cuda.is_available()` returns False after installing pyannote/whisperx.

### Pitfall 2: ctranslate2 CUDA version mismatch (faster-whisper)
**What goes wrong:** `This CTranslate2 package was not compiled with CUDA support` error at runtime.
**Why it happens:** ctranslate2 >= 4.5.0 requires CUDA 12 + cuDNN 9. Older versions only support cuDNN 8.
**How to avoid:** Use `pip install faster-whisper` without version pinning — the latest ctranslate2 (CUDA 12 + cuDNN 9) will be installed automatically. Confirm with `ctranslate2.cuda_device_count()`.
**Warning signs:** Runs on CPU despite `device="cuda"` setting.

### Pitfall 3: Parakeet processes MP4 directly
**What goes wrong:** `nemo_asr.models.ASRModel.transcribe(['video.mp4'])` fails or produces garbage.
**Why it happens:** Parakeet TDT expects 16kHz mono WAV or FLAC. MP4/H.264 containers are not accepted.
**How to avoid:** Extract WAV from MP4 using ffmpeg before passing to Parakeet (same pattern already used in `diarize.py`). Delete the temp WAV after transcription.
**Warning signs:** Silent failure or NeMo codec error on MP4 input.

### Pitfall 4: Parakeet confidence field missing
**What goes wrong:** Downstream code reads `word["confidence"]` and gets KeyError or None.
**Why it happens:** Parakeet TDT does not expose per-word probability like faster-whisper's `w.probability`. The NeMo output object for word timestamps includes only `start`, `end`, and `word`.
**How to avoid:** Set `"confidence": 1.0` (constant) in the output words dict. D-11 allows new fields but existing fields must be present — do not omit `confidence`.
**Warning signs:** Frontend `subtitleStore.ts` type errors or undefined confidence values in the editor.

### Pitfall 5: NeMo MSDD requires YAML config and manifest files
**What goes wrong:** NeMo diarization is not a simple `pipeline.from_pretrained()` call — it requires an OmegaConf YAML config file and an audio manifest JSON pointing to input files.
**Why it happens:** NeMo's diarization pipeline is designed for batch research workflows, not single-file subprocess scripts.
**How to avoid:** Either (a) use NeMo's `ClusteringDiarizer` with inline config construction (no YAML file required), or (b) fall back to pyannote if the config complexity exceeds the integration budget.
**Warning signs:** Import errors on `hydra`, `omegaconf`, or YAML file not found errors.

### Pitfall 6: NeMo MSDD requires separate audio manifest file
**What goes wrong:** MSDD inference requires a manifest.json with audio file paths written to disk before running.
**Why it happens:** NeMo batch processing paradigm — all inputs go through a manifest file.
**How to avoid:** Write a temporary manifest.json in the temp dir before diarization, clean up after. This is extra I/O but not architecturally problematic.
**Warning signs:** `FileNotFoundError` for manifest file.

### Pitfall 7: Word timestamp format difference between Parakeet and faster-whisper
**What goes wrong:** transcript.json words are malformed because the field names differ.
**Why it happens:** faster-whisper uses `.word`, `.start`, `.end`, `.probability`; Parakeet uses `{'word': str, 'start': float, 'end': float}` with no probability.
**How to avoid:** Map Parakeet output explicitly. Do not assume field parity.

### Pitfall 8: `just` shebang and venv on Windows
**What goes wrong:** The `dev` recipe uses `#!/usr/bin/env bash` which may not find Git Bash on some Windows setups.
**Why it happens:** `just` on Windows uses sh from Git for Windows, but the path can vary.
**How to avoid:** Keep the shebang but test on the target machine. The simple recipes (setup-python, backend, frontend) don't need the shebang — only `dev` does.

## Code Examples

### Parakeet TDT transcription (full script pattern)
```python
# Source: https://huggingface.co/nvidia/parakeet-tdt-0.6b-v2
import sys, json, os, subprocess, tempfile
import nemo.collections.asr as nemo_asr

def main():
    audio_path = sys.argv[1]   # .mp4 or .wav
    output_path = sys.argv[2]
    language = sys.argv[3] if len(sys.argv) > 3 else "en"

    print(json.dumps({"type": "progress", "percent": 0, "status": "loading_model"}), flush=True)

    asr_model = nemo_asr.models.ASRModel.from_pretrained("nvidia/parakeet-tdt-0.6b-v2")
    asr_model = asr_model.to("cuda")
    asr_model.eval()

    # Parakeet requires WAV input — extract from mp4 if needed
    wav_path = audio_path
    cleanup_wav = False
    if audio_path.lower().endswith(".mp4"):
        wav_path = os.path.join(tempfile.gettempdir(), ".parakeet_temp.wav")
        subprocess.run(
            ["ffmpeg", "-y", "-i", audio_path, "-vn", "-acodec", "pcm_s16le",
             "-ar", "16000", "-ac", "1", wav_path],
            capture_output=True, check=True
        )
        cleanup_wav = True

    print(json.dumps({"type": "progress", "percent": 10, "status": "transcribing"}), flush=True)

    output = asr_model.transcribe([wav_path], timestamps=True)

    if cleanup_wav and os.path.exists(wav_path):
        os.remove(wav_path)

    word_timestamps = output[0].timestamp['word']

    words = []
    for stamp in word_timestamps:
        words.append({
            "word": stamp['word'].strip(),
            "start": round(stamp['start'], 3),
            "end": round(stamp['end'], 3),
            "confidence": 1.0,  # Parakeet has no per-word probability
        })

    transcript = {"language": language, "words": words}
    with open(output_path, "w") as f:
        json.dump(transcript, f, indent=2)

    print(json.dumps({"type": "done", "percent": 100, "language": language, "word_count": len(words)}), flush=True)

if __name__ == "__main__":
    main()
```

## Open Questions

1. **Does `triton-windows` + `nemo_toolkit[asr]` install cleanly on Python 3.12 / CUDA 13.1?**
   - What we know: triton-windows 3.6.0 supports Python 3.10–3.14; nemo_toolkit 2.7.2 officially doesn't support Windows
   - What's unclear: Whether the triton-windows workaround resolves ALL Windows-blocking deps (there may be others beyond triton)
   - Recommendation: Spike Task 1 resolves this — fail fast if more than 1-2 dependency hacks are needed

2. **Does Parakeet TDT word timestamp precision match faster-whisper large-v3?**
   - What we know: Parakeet outputs `timestamp['word']` with start/end; faster-whisper uses CTC alignment
   - What's unclear: Actual precision for subtitle use (karaoke highlight); Parakeet benchmarks focus on WER, not timestamp accuracy
   - Recommendation: Spike test with a known audio clip; compare word boundary timing visually in the existing editor

3. **NeMo MSDD integration complexity — is it worth it?**
   - What we know: MSDD requires OmegaConf config, manifest JSON, and output directory; pyannote requires one Pipeline.from_pretrained() call
   - What's unclear: Whether MSDD provides measurably better diarization on the typical 2-speaker video use case
