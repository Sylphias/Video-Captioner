    // Animation scope: phrase-level (whole phrase animates together) or word-level (each word animates independently)

    // Enter/exit animation types
    export type EnterExitType = 'none' | 'fade' | 'slide-up' | 'slide-down' | 'slide-left' | 'slide-right'

    // Active (hold) animation types â€” loops while phrase is on screen
    export type ActiveType = 'none' | 'jiggle' | 'wave' | 'pulse' | 'bounce'

    // Easing options â€” maps to Remotion Easing functions
    export type EasingType = 'linear' | 'ease-in' | 'ease-out' | 'ease-in-out'
      | 'ease-in-cubic' | 'ease-out-cubic' | 'ease-in-out-cubic'
      | 'bounce' | 'elastic' | 'back' | 'spring'

    // Per-phase configuration for enter or exit
    export interface AnimationPhaseConfig {
      type: EnterExitType
      durationSec: number         // e.g. 0.3
      easing: EasingType
      params: Record<string, number>  // type-specific params (slideOffsetFraction, staggerFrames, etc.)
    }

    // Active phase configuration
    export interface ActivePhaseConfig {
      type: ActiveType
      cycleDurationSec: number    // duration of one loop cycle
      intensity: number           // 0-1 amplitude/strength multiplier

      id: string
      name: string
      isBuiltin: boolean
      scope: AnimationScope
      enter: AnimationPhaseConfig
      active: ActivePhaseConfig
      exit: AnimationPhaseConfig & { mirrorEnter: boolean }  // exit can mirror enter by default
      createdAt: number
      updatedAt: number
    }

    **Create packages/remotion-composition/src/animations.ts** â€” pure functions, no React hooks, no side effects.

    This module provides two main exports:

    1. `computeAnimationStyles(phraseStartSec, phraseEndSec, preset, frame, fps, width, height): React.CSSProperties` â€” for phrase-scope animations. Computes enter/active/exit progress and returns CSS transform/opacity/filter styles.

    2. `computeWordAnimationStyles(wordIndex, wordCount, phraseStartSec, phraseEndSec, preset, frame, fps, width, height): React.CSSProperties` â€” for word-scope animations. Adds per-word stagger offset.

    Implementation details:

    **Easing mapper** â€” `getEasingFn(easing: EasingType): (t: number) => number` maps EasingType strings to Remotion Easing functions:
    - `'linear'` -> `Easing.linear`
    - `'ease-in'` -> `Easing.in(Easing.ease)`
    - `'ease-out'` -> `Easing.out(Easing.ease)`
    - `'ease-in-out'` -> `Easing.inOut(Easing.ease)`
    - `'ease-in-cubic'` -> `Easing.in(Easing.cubic)`
    - `'ease-out-cubic'` -> `Easing.out(Easing.cubic)`
    - `'ease-in-out-cubic'` -> `Easing.inOut(Easing.cubic)`
    - `'bounce'` -> `Easing.bounce`
    - `'elastic'` -> `Easing.elastic(1)`
    - `'back'` -> `Easing.back(1.5)`
    - `'spring'` -> `Easing.out(Easing.ease)` (spring uses the `spring()` function directly, not easing â€” fall back to ease-out)

    **Enter phase computation** â€” `applyEnterAnimation(progress: number, config: AnimationPhaseConfig, width: number, height: number): React.CSSProperties`:
    - `'none'` -> `{}`
    - `'fade'` -> `{ opacity: progress }`
    - `'slide-up'` -> offset = `(config.params.slideOffsetFraction ?? 0.15) * height`; `{ opacity: progress, transform: translateY(offset * (1-progress))px }`
    - `'slide-down'` -> same but negative Y offset
    - `'slide-left'` -> X offset with width
    - `'slide-right'` -> negative X offset with width
    - `'pop'` -> use `spring()` with frame-based input: `{ transform: scale(springProgress), opacity: springProgress > 0.01 ? 1 : 0 }`. Spring config: stiffness 180, damping 12, overshootClamping false.
    - `'bounce'` -> use `spring()` with stiffness 260, damping 8 â€” more pronounced bounce
    - `'fly-in'` -> larger slide offset (0.5 of width) + opacity
    - `'shrink'` -> `{ transform: scale(1 + (1 - progress) * 0.5), opacity: progress }`
    - `'typewriter'` -> return `{ __typewriterProgress: progress }` as a special marker (handled in SubtitleOverlay text slicing, not as CSS)
    - `'letter-by-letter'` -> similar to typewriter but per-character
    - `'word-cascade'` -> handled by word-scope path
    - `'blur-reveal'` -> `{ filter: blur(${(1-progress) * 8}px), opacity: interpolate(progress, [0, 0.3], [0, 1], { extrapolateRight: 'clamp' }) }`

    **Exit phase computation** â€” `applyExitAnimation(progress: number, config, width, height)`:
    - If `config.mirrorEnter`, invert the enter animation (progress goes 1->0 conceptually, so pass `1 - progress` to applyEnterAnimation).
    - Otherwise, apply exit-specific type with same mapping as enter but reversed direction.

    **Active phase computation** â€” `applyActiveAnimation(activeFrame: number, fps: number, config: ActivePhaseConfig, height: number)`:
    - `'none'` -> `{}`
    - `'jiggle'` -> small random-like rotation using `Math.sin(t * 4 * Math.PI)` for fast oscillation: `{ transform: rotate(${angle}deg) }` where angle = intensity * 3 * sin
    - `'wave'` -> `{ transform: translateY(${sin(t * 2PI) * intensity * height * 0.02}px) }`
    - `'pulse'` -> `{ transform: scale(${1 + sin(t * 2PI) * intensity * 0.05}) }`
    - `'bounce'` -> vertical bounce using `Math.abs(Math.sin(t * Math.PI))`: `{ transform: translateY(${-absVal * intensity * height * 0.02}px) }`
    - `t` is computed as `(activeFrame % cycleFrames) / cycleFrames` where `cycleFrames = Math.round(config.cycleDurationSec * fps)`

    **Frame clamping** (Pitfall #2 from research): At the top of `computeAnimationStyles`, clamp enter+exit durations:
    ```typescript
    const totalPhraseFrames = Math.round((phraseEndSec - phraseStartSec) * fps)
    const maxEnterFrames = Math.floor(totalPhraseFrames * 0.45)
    const maxExitFrames = Math.floor(totalPhraseFrames * 0.45)
    const enterFrames = Math.min(Math.round(preset.enter.durationSec * fps), maxEnterFrames)
    const exitFrames = Math.min(Math.round(preset.exit.durationSec * fps), maxExitFrames)
    ```

    **Word-scope stagger** for `computeWordAnimationStyles`:
    - The stagger delay per word: `staggerFrames = config.params.staggerFrames ?? 3`
    - Each word's effective frameIntoPhrase is offset: `Math.max(0, frameIntoPhrase - wordIndex * staggerFrames)`
    - Otherwise computes the same enter/active/exit as phrase scope

    For `'typewriter'` and `'letter-by-letter'` enter types, return a special `__textSliceProgress` property (number 0-1) in the CSSProperties (cast as any). The SubtitleOverlay will read this to clip text. This is cleaner than mixing text logic into the pure animation helpers.

    **In packages/remotion-composition/src/SubtitleOverlay.tsx**, integrate the animation system:

    - Import `computeAnimationStyles`, `computeWordAnimationStyles` from `'./animations'`
    - Import `AnimationPreset` type from `@eigen/shared-types`
    - Add `animationPreset?: AnimationPreset` to `SubtitleOverlayProps`
    - In the phrase rendering loop, after computing `effectiveStyle`:
      1. Determine the phrase's animation preset: check `phrase.animationPresetId` (per-phrase override from shared-types, but for now just use the global `animationPreset` prop â€” per-phrase overrides will be wired in Plan 03)
      2. If no preset or preset scope is 'phrase': call `computeAnimationStyles(phraseStart, phraseEnd, preset, frame, fps, width, height)` and spread the result onto `containerStyle`.
      3. If preset scope is 'word': for each word span, call `computeWordAnimationStyles(wordIndex, wordCount, ...)` and apply to the individual `<span>` style.
      4. Handle typewriter: if the returned style has `__textSliceProgress`, slice the displayed text to `Math.floor(progress * fullText.length)` characters and render only that.
    - Add `width` and `height` from `useVideoConfig()` (already available via the hook).
    - **Important:** The animation styles must be applied to BOTH the stroke layer and fill layer spans for consistency. When using phrase-scope animation, apply transform/opacity to the phrase container div. When word-scope, apply to each word span individually (both stroke and fill layers).
    - When there is no animationPreset (or preset is undefined), render exactly as before (no animation) â€” backward compatible.
