    **Create packages/backend/src/services/animationPresets.ts:**

    Follow the existing jobStore.ts Fastify plugin pattern using `fp()`.

    ```typescript
    import fp from 'fastify-plugin'
    import type { FastifyInstance } from 'fastify'
    import Database from 'better-sqlite3'
    import path from 'node:path'
    import crypto from 'node:crypto'

      }
    }
    ```

    The plugin should:
    1. Open `path.join(DATA_ROOT, 'presets.db')` with WAL journal mode.
    2. Create the `animation_presets` table if not exists:
       ```sql
       CREATE TABLE IF NOT EXISTS animation_presets (
         id TEXT PRIMARY KEY,
         name TEXT NOT NULL,
         is_builtin INTEGER NOT NULL DEFAULT 0,
         scope TEXT NOT NULL DEFAULT 'phrase',
         params TEXT NOT NULL,
         created_at INTEGER NOT NULL,
         updated_at INTEGER NOT NULL
       )
       ```
    3. Decorate fastify with `db`.
    4. Register `onClose` hook to close the database.
    5. After table creation, call `seedBuiltinPresets(db)`.

    **Built-in presets seeding** — define `BUILTIN_PRESETS` as a const array of 7 presets (Claude's Discretion per CONTEXT.md). For each preset, check if its ID exists in the DB; if not, insert it. This means built-in presets survive even if the user adds custom ones, and are re-seeded if the DB is recreated.

    Use deterministic IDs for built-in presets: `builtin-classic-fade`, `builtin-slide-up`, `builtin-pop-in`, `builtin-word-cascade`, `builtin-typewriter`, `builtin-blur-reveal`, `builtin-jiggle-pop`.

    **The 7 built-in presets (per research recommendations):**

    1. **Classic Fade** (scope: phrase) — Enter: fade, 0.3s, ease-out. Active: none. Exit: mirror enter (fade, 0.2s, ease-in).
    2. **Slide Up** (scope: phrase) — Enter: slide-up, 0.25s, ease-out-cubic, slideOffsetFraction: 0.15. Active: none. Exit: mirror enter.
    3. **Pop In** (scope: phrase) — Enter: pop, 0.35s, spring. Active: none. Exit: shrink, 0.2s, ease-in. mirrorEnter: false.
    4. **Word Cascade** (scope: word) — Enter: word-cascade, 0.4s, ease-out-cubic, staggerFrames: 3. Active: none. Exit: fade, 0.2s, ease-in. mirrorEnter: false.
    5. **Typewriter** (scope: phrase) — Enter: typewriter, 0.8s, linear. Active: pulse (cursor blink effect), cycleDurationSec: 0.5, intensity: 0.3. Exit: fade, 0.15s, ease-in. mirrorEnter: false.
    6. **Blur Reveal** (scope: phrase) — Enter: blur-reveal, 0.4s, ease-out. Active: none. Exit: mirror enter.
    7. **Jiggle Pop** (scope: phrase) — Enter: pop, 0.3s, spring. Active: jiggle, cycleDurationSec: 0.6, intensity: 0.5. Exit: fade, 0.15s, ease-in. mirrorEnter: false.

    Each preset params blob is a JSON object matching the shape:
    ```json
    {
      "enter": { "type": "...", "durationSec": ..., "easing": "...", "params": { ... } },
      "active": { "type": "...", "cycleDurationSec": ..., "intensity": ... },
      "exit": { "mirrorEnter": true/false, "type": "...", "durationSec": ..., "easing": "...", "params": { ... } }
    }
    ```

    The `scope` is stored as a separate column (not inside params) for potential future querying.

    **Create packages/backend/src/routes/presets.ts:**

    Follow the existing route plugin pattern (see routes/upload.ts, routes/render.ts). Use `fp()` wrapper.

    **Routes:**

    1. `GET /api/presets` — Returns all presets as JSON array. Each row's `params` field is parsed from JSON string to object. Response shape: `AnimationPreset[]` (matching the shared-types interface).
       - Map DB rows to response: `{ id, name, isBuiltin: Boolean(row.is_builtin), scope: row.scope, ...JSON.parse(row.params), createdAt: row.created_at, updatedAt: row.updated_at }`
       - Order by `created_at ASC` (built-ins first since they have earliest timestamps).

    2. `POST /api/presets` — Create a new preset. Body: `{ name: string, scope: string, enter: {...}, active: {...}, exit: {...} }`.
       - Generate ID: `crypto.randomUUID()`
       - Store params as JSON.stringify of `{ enter, active, exit }`
       - Set `is_builtin = 0`, `created_at = Date.now()`, `updated_at = Date.now()`
       - Return the created preset (same shape as GET items) with 201 status.

    3. `PUT /api/presets/:id` — Update an existing preset. Body: `{ name?, scope?, enter?, active?, exit? }`.
       - Fetch existing row first. If not found, return 404.
       - If `is_builtin = 1`, return 403 with error "Cannot modify built-in presets".
       - Merge provided fields with existing params (parse existing params JSON, spread new fields).
       - Update `updated_at = Date.now()`.
       - Return the updated preset.

    4. `DELETE /api/presets/:id` — Delete a preset.
       - Fetch existing row. If not found, return 404.
       - If `is_builtin = 1`, return 403 with error "Cannot delete built-in presets".
       - Delete the row. Return `{ ok: true }`.

    5. `POST /api/presets/:id/duplicate` — Duplicate a preset (creates a copy with a new name).
       - Fetch source row. If not found, return 404.
       - Create new row with `crypto.randomUUID()`, name: `${sourceName} (Copy)`, `is_builtin = 0`.
       - Copy scope and params from source.
       - Return the new preset with 201 status.

    Use prepared statements for all queries (better-sqlite3 best practice — `db.prepare(sql)`). The prepared statements can be created inside the plugin function body (they're bound to the db instance).

    **Register in packages/backend/src/index.ts:**

    Add imports:
    ```typescript
    import animationPresetsPlugin from './services/animationPresets.ts'
    import presetsRoutes from './routes/presets.ts'
    ```

    Register the plugin BEFORE routes (after jobStorePlugin, before route registrations):
    ```typescript
    await fastify.register(animationPresetsPlugin)
    ```

    Register the routes (after other route registrations):
    ```typescript

    ```

    **Important:** The animationPresetsPlugin MUST be registered before presetsRoutes because routes depend on `fastify.db` being decorated.
