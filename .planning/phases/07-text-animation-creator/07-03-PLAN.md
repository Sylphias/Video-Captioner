    **Create packages/frontend/src/hooks/useAnimationPresets.ts:**

    A data-fetching hook for animation presets. Pattern: useState + useEffect + fetch (no external data library -- matches existing useUpload/useTranscribe pattern).

    ```typescript
    import { useState, useEffect, useCallback } from 'react'
    import type { AnimationPreset } from '@eigen/shared-types'

    interface UseAnimationPresetsReturn {
      presets: AnimationPreset[]
      loading: boolean
      error: string | null
      refresh: () => void
      createPreset: (preset: Omit<AnimationPreset, 'id' | 'isBuiltin' | 'createdAt' | 'updatedAt'>) => Promise<AnimationPreset>
      updatePreset: (id: string, patch: Partial<Pick<AnimationPreset, 'name' | 'scope' | 'enter' | 'active' | 'exit'>>) => Promise<AnimationPreset>
      deletePreset: (id: string) => Promise<void>
      duplicatePreset: (id: string) => Promise<AnimationPreset>
    }
    ```

    Implementation:
    - `presets` state initialized to empty array.
    - `loading` starts true, set false after fetch completes.
    - `fetchPresets()`: `GET /api/presets`, parse JSON, set presets. Called on mount and on `refresh()`.
    - `createPreset()`: `POST /api/presets` with JSON body, returns created preset, auto-refreshes list.
    - `updatePreset()`: `PUT /api/presets/${id}` with JSON body, returns updated preset, auto-refreshes.
    - `deletePreset()`: `DELETE /api/presets/${id}`, auto-refreshes.
    - `duplicatePreset()`: `POST /api/presets/${id}/duplicate`, returns new preset, auto-refreshes.
    - All mutation functions should call `refresh()` after success to keep the list in sync.
    - Error handling: catch fetch errors, set `error` state. Log but don't throw.

    **In packages/frontend/src/components/PreviewPanel.tsx:**

    The PreviewPanel builds `inputProps` for the Remotion Player. It needs to resolve BOTH the global active animation preset ID AND per-phrase animation preset IDs to full `AnimationPreset` objects before passing them across the Remotion serialization boundary. The Remotion composition receives props only -- it cannot access hooks, stores, or APIs.

    1. Import `useAnimationPresets` hook.
    2. Call `const { presets } = useAnimationPresets()` at the top of the component.
    3. Import `useSubtitleStore` selectors for `activeAnimationPresetId` and `phraseAnimationPresetIds`.
    4. In the `inputProps` useMemo, resolve BOTH the global preset AND per-phrase presets:
       ```typescript
       const activeAnimationPresetId = useSubtitleStore((s) => s.activeAnimationPresetId)
       const phraseAnimationPresetIds = useSubtitleStore((s) => s.phraseAnimationPresetIds)

       const inputProps = useMemo(() => {
         if (!jobId || !session) return null
         const videoSrc = `/api/jobs/${jobId}/video`

         // Resolve the global active animation preset from the presets list
         const animationPreset = activeAnimationPresetId
           ? presets.find(p => p.id === activeAnimationPresetId) ?? undefined
           : undefined

         // CRITICAL: Resolve per-phrase animation preset IDs to full AnimationPreset objects.
         // The Remotion composition runs in a serialization boundary and CANNOT access
         // hooks/stores/APIs. Per-phrase presets MUST be resolved here in the frontend
         // and passed as full AnimationPreset objects on each phrase entry.
         const phrases = session.phrases.map((p, i) => {
           const phraseOverrideId = phraseAnimationPresetIds[i]
           const phraseAnimationPreset = phraseOverrideId
             ? presets.find(pr => pr.id === phraseOverrideId) ?? undefined
             : undefined

           return {
             words: p.words,
             dominantSpeaker: p.dominantSpeaker,
             lingerDuration: p.lingerDuration,
             styleOverride: p.styleOverride,
             // Per-phrase resolved AnimationPreset (full object, not just an ID).
             // SubtitleOverlay will use: phrase.animationPreset ?? globalAnimationPreset
             animationPreset: phraseAnimationPreset,
           }
         })

         return {
           videoSrc,
           phrases,  // CompositionPhrase[] with resolved per-phrase animationPreset
           style,
           speakerStyles,
           animationPreset,  // global default AnimationPreset
         }
       }, [jobId, session, style, speakerStyles, activeAnimationPresetId, phraseAnimationPresetIds, presets])
       ```

    5. Add `activeAnimationPresetId`, `phraseAnimationPresetIds`, and `presets` to the `useMemo` dependency array.
