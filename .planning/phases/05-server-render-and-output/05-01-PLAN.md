**1. Create render service** (`packages/backend/src/services/render.ts`):

This module has two responsibilities: (a) bundle the Remotion composition once at startup, (b) dispatch render jobs to a worker thread.

```typescript
import { bundle } from '@remotion/bundler'
import { Worker } from 'node:worker_threads'
import path from 'node:path'
import { fileURLToPath } from 'node:url'
import { updateJob } from './jobStore.ts'
import type { Job } from '@eigen/shared-types'

const __dirname = path.dirname(fileURLToPath(import.meta.url))

let bundleLocation: string | null = null

export async function initBundle(): Promise<void> {
  const entryPoint = path.resolve(__dirname, '../../../remotion-composition/src/remotion-entry.ts')
  bundleLocation = await bundle({ entryPoint })
  console.log(`[render] Remotion bundle ready at ${bundleLocation}`)
}

export function getBundleLocation(): string {
  if (!bundleLocation) throw new Error('Remotion bundle not initialized — call initBundle() first')
  return bundleLocation
}
```

For `dispatchRender`, the function:
- Takes `jobs: Map<string, Job>`, `jobId: string`, and `outputPath: string`
- Reads the job's metadata (fps, width, height, duration) and session phrases from the request
- Creates a `Worker` pointing to the render-worker file
- Passes `workerData` with: `bundleLocation`, `outputPath`, `compositionId` (from COMPOSITION_ID), `inputProps` (videoSrc as `http://localhost:3001/api/jobs/${jobId}/video`, phrases, style), `durationInFrames` (`Math.floor(duration * fps)`), `fps`, `width`, `height`
- Listens for `message` events: on `{ type: 'progress', progress }` → `updateJob(jobs, jobId, { progress })`, on `{ type: 'done' }` → `updateJob(jobs, jobId, { status: 'rendered', progress: 100, outputPath })`
- Listens for `error` event → `updateJob(jobs, jobId, { status: 'failed', error: err.message })`
- Listens for `exit` event with non-zero code → set failed status if not already rendered

For the worker file path: use `new URL('../workers/render-worker.ts', import.meta.url)` since Node.js `--experimental-strip-types` can load `.ts` worker files via URL. If this doesn't work at runtime, fall back to pointing at the `.ts` file via `path.resolve(__dirname, '../workers/render-worker.ts')`. Test this during implementation with a quick verification.

**2. Create render worker** (`packages/backend/src/workers/render-worker.ts`):

First create the directory: `mkdir -p packages/backend/src/workers`

The worker:
- Imports `parentPort` and `workerData` from `node:worker_threads`
- Imports `renderMedia` and `selectComposition` from `@remotion/renderer`
- Destructures from `workerData`: `bundleLocation`, `outputPath`, `compositionId`, `inputProps`, `durationInFrames`, `fps`, `width`, `height`
- Calls `selectComposition({ serveUrl: bundleLocation, id: compositionId, inputProps })`
- Overrides the composition with actual video metadata: `{ ...composition, durationInFrames, fps, width, height }`
- Calls `renderMedia({ composition: resolvedComposition, serveUrl: bundleLocation, codec: 'h264', outputLocation: outputPath, inputProps, onProgress: ({ progress }) => parentPort?.postMessage({ type: 'progress', progress: Math.round(progress * 100) }) })`
- On successful completion: `parentPort?.postMessage({ type: 'done' })`
- Wrap the entire thing in a try/catch — on error, `parentPort?.postMessage({ type: 'error', error: err.message })` and `process.exit(1)`

**3. Create render routes** (`packages/backend/src/routes/render.ts`):

Wrap in `fp()` from `fastify-plugin` (same pattern as all other route files).

POST `/api/jobs/:jobId/render`:
- Extract `jobId` from params
- Extract `phrases` and `style` from request body (JSON). These are the current session phrases and style from the frontend store. Type the body as `{ phrases: TranscriptPhrase[], style: StyleProps }` (import from shared-types and remotion-composition types respectively). For simplicity, define the body shape inline or use a minimal interface.
- Validate: job exists, job status is 'transcribed' (only render after transcription is done — diarization returns to 'transcribed' so this covers that case too), job has metadata
- Set `outputPath = path.join(DATA_ROOT, jobId, 'output.mp4')`
- Update job: `status: 'rendering', progress: 0, outputPath`
- Call `dispatchRender(fastify.jobs, jobId, outputPath, inputProps)` where inputProps includes `videoSrc: \`http://localhost:3001/api/jobs/${jobId}/video\``, the phrases, and the style
- Return `202 Accepted` with `{ jobId, status: 'rendering' }`

GET `/api/jobs/:jobId/download`:
- Extract jobId from params
- Validate: job exists, status is 'rendered', outputPath is set
- Verify the file exists on disk (`access(outputPath)`)
- Set headers: `Content-Disposition: attachment; filename="output-${jobId}.mp4"`, `Content-Type: video/mp4`
- Stream the file: `reply.send(createReadStream(job.outputPath))`
- If file not found, return 404
