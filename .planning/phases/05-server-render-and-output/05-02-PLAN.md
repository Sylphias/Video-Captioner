**1. Create `useRender` hook** (`packages/frontend/src/hooks/useRender.ts`):

Follow the same pattern as `useTranscribe` — a focused state machine hook for the render lifecycle.

State interface:
```typescript
interface RenderState {
  status: 'idle' | 'rendering' | 'rendered' | 'failed'
  progress: number    // 0-100
  error?: string
}
```

The hook exports:
- `state: RenderState` — current render state
- `render(jobId: string): void` — triggers the render
- `reset(): void` — resets to idle

`render(jobId)` implementation:
1. Set state to `{ status: 'rendering', progress: 0 }`
2. Read current phrases and style from the Zustand store: `const { session, style } = useSubtitleStore.getState()`. Extract `phrases` from `session.phrases` — map each SessionPhrase to `{ words: p.words }` (strip isManualSplit/dominantSpeaker, same as PreviewPanel's inputProps mapping). Import `useSubtitleStore` from the store.
3. POST to `/api/jobs/${jobId}/render` with JSON body `{ phrases, style }`. If the POST fails (non-2xx), set failed state with error message and return.
4. After successful POST, open an `EventSource` to `/api/jobs/${jobId}/status` for SSE progress tracking.
5. On SSE message: parse the JSON data. If `status === 'rendering'`, update progress from `data.progress`. If `status === 'rendered'`, set state to `{ status: 'rendered', progress: 100 }` and close the EventSource. If `status === 'failed'`, set failed state with `data.error` and close.
6. On SSE error: set failed state and close.

`reset()`: Set state back to `{ status: 'idle', progress: 0 }`.

Clean up: Close the EventSource on unmount (use `useRef` to hold the EventSource, clean up in `useEffect` return).

**2. Integrate render/download UI into SubtitlesPage:**

In the `transcribed` state section of SubtitlesPage (where the preview, editor, and controls are shown):

Add a render controls section in the `subtitles-page__top-controls` area (alongside existing "Go to subtitle" and diarize controls). Place it as a new group after the diarize controls:

- **Render button:** Triggers `render(uploadState.jobId!)`. Disabled while `renderState.status === 'rendering'`. Label:
  - idle: "Render MP4"
  - rendering: "Rendering... {progress}%"
  - rendered: "Re-render MP4"
  - failed: "Retry render"

- **Render progress bar:** Show only when `renderState.status === 'rendering'`. Use the same progress bar pattern as upload/transcription (`.subtitles-page__progress-track` + `.subtitles-page__progress-fill`). Place it below the render button in the controls area.

- **Download button:** Show only when `renderState.status === 'rendered'`. This is an `<a>` tag (not a button) pointing to `/api/jobs/${uploadState.jobId}/download` with `download` attribute. Style it as a prominent green button. Label: "Download MP4".

- **Render error:** Show inline below the render controls when `renderState.status === 'failed'`. Same pattern as diarize error.

Wire up: Import and call `useRender()` in SubtitlesPage alongside the other hooks. Add `resetRender` to the `resetAll` function.
