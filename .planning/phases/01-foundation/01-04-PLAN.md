    **`packages/backend/src/routes/upload.ts`:**
    Export a Fastify plugin that registers `POST /api/upload`:
    1. Call `req.file()` to get the multipart file data
    2. If no file, return 400 `{ error: 'No file provided' }`
    3. Generate a job ID using `crypto.randomUUID()` (native Node, no uuid package needed — but uuid is already installed, use whichever is cleaner)
    4. Create job directory: `path.join(DATA_ROOT, jobId)`; `mkdir` recursive
    5. Create job in store with status `'uploading'`, progress 0, originalFilename from `data.filename`
    6. Stream file to `{jobDir}/original{ext}` using `pipeline(data.file, createWriteStream(dest))` — NEVER buffer in memory
    7. After stream completes, reply `202 { jobId }` immediately
    8. Fire-and-forget async normalization pipeline:
       a. Update job status to `'normalizing'`, progress 0
       b. Call `probeVideo(originalPath)` to get duration for progress calculation
       c. Call `normalizeVideo(originalPath, path.join(jobDir, 'normalized.mp4'), onProgress)` — the `onProgress` callback updates job.progress (0-100)
       d. After normalization, call `probeVideo(path.join(jobDir, 'normalized.mp4'))` to get metadata from the normalized file
       e. Call `extractThumbnail(path.join(jobDir, 'normalized.mp4'), path.join(jobDir, 'thumbnail.jpg'))` — extract from normalized video (per research recommendation, consistent frame timing)
       f. Update job: status `'ready'`, metadata set, thumbnailPath set
       g. On any error: update job status to `'failed'`, set error message
    - CRITICAL: The original file is preserved untouched. Normalization creates a SEPARATE `normalized.mp4`. Per user decision: two files per job.
    - IMPORTANT: Do not try to detect VFR — always normalize (per Pitfall 7 in research).

    **`packages/backend/src/routes/jobs.ts`:**
    Export a Fastify plugin that registers:

    `GET /api/jobs/:jobId/status` — Server-Sent Events endpoint:
    - Set response headers: `Content-Type: text/event-stream`, `Cache-Control: no-cache`, `Connection: keep-alive`
    - Use `reply.raw` to write SSE directly (if @fastify/sse proves difficult with the exact API, fall back to manual SSE writing on `reply.raw` — the research notes that @fastify/sse API shape has medium confidence)
    - Poll the job store every 500ms and send the job state as an SSE `data:` event (JSON-encoded Job object)
    - When job status is `'ready'` or `'failed'`, send a final event and close the connection
    - Clean up interval on client disconnect (`req.raw.on('close', ...)`)

    `GET /api/jobs/:jobId/thumbnail` — Serve the thumbnail image:
    - Read job from store, check thumbnailPath exists
    - If no thumbnail or job not found, return 404
    - Use `reply.sendFile` or `reply.send(createReadStream(thumbnailPath))` with `Content-Type: image/jpeg`
    - This endpoint lets the frontend display the thumbnail after upload completes

    **Update `packages/backend/src/index.ts`:**
    Import and register both route plugins (`uploadRoutes`, `jobRoutes`) with `/api` prefix. Ensure they are registered AFTER the plugins and job store decorator.

    Note on SSE implementation: If `@fastify/sse` has API issues at runtime, implement SSE manually — it is straightforward:
    ```typescript
    reply.raw.writeHead(200, {
      'Content-Type': 'text/event-stream',

      'Connection': 'keep-alive'
    })
    // To send: reply.raw.write(`data: ${JSON.stringify(job)}\n\n`)
    // To close: reply.raw.end()
    ```
    Manual SSE is more reliable than depending on a plugin with uncertain API shape.

    **`packages/frontend/src/hooks/useUpload.ts`:**
    Custom hook managing the full upload lifecycle. Returns state and a trigger function.

    ```typescript
    interface UploadState {
      status: 'idle' | 'uploading' | 'normalizing' | 'ready' | 'failed'
      progress: number      // 0-100
      jobId: string | null
      job: Job | null       // full job object from SSE (includes metadata)
      error: string | null
    }
    ```

    `useUpload()` returns `{ state: UploadState, upload: (file: File) => void, reset: () => void }`:

    `upload(file)`:
    1. Set status to 'uploading', progress to 0
    2. Create FormData with the file, POST to `/api/upload`
       - Use XMLHttpRequest (not fetch) for upload progress: `xhr.upload.onprogress` gives bytes sent/total for the "Uploading... 45%" display
       - On success (202), parse `{ jobId }` from response
    3. Set jobId, then connect to SSE: `new EventSource(/api/jobs/${jobId}/status)`
       - On each SSE message: parse the Job JSON, update state (status, progress, metadata)
       - When status is 'normalizing': update progress from job.progress
       - When status is 'ready': close EventSource, set final state with job metadata
       - When status is 'failed': close EventSource, set error from job.error
    4. Clean up EventSource on unmount (return cleanup from useEffect)

    `reset()`: Set state back to idle, clear jobId/job/error.
