    **1. Create `useBuilderStore.ts`** — local Zustand store for Animation Builder state:

    ```typescript
    import { create } from 'zustand'
    import type { KeyframeTrack, KeyframeEasing, AnimationPreset, KeyframeableProperty } from '@eigen/shared-types'

    type AspectRatio = '16:9' | '9:16' | '1:1'

    interface BuilderState {
      // Preset being edited (loaded from API or new)
      preset: AnimationPreset | null
      setPreset: (p: AnimationPreset | null) => void

      // Keyframe tracks (working copy)
      keyframeTracks: KeyframeTrack[]
      setKeyframeTracks: (tracks: KeyframeTrack[]) => void
      addKeyframe: (property: KeyframeableProperty, time: number, value: number) => void
      removeKeyframe: (property: KeyframeableProperty, index: number) => void
      updateKeyframeValue: (property: KeyframeableProperty, index: number, value: number) => void
      updateKeyframeTime: (property: KeyframeableProperty, index: number, time: number) => void
      setTrackEasing: (property: KeyframeableProperty, segmentIndex: number, easing: KeyframeEasing) => void

      // Preview state
      aspectRatio: AspectRatio
      setAspectRatio: (ar: AspectRatio) => void
      sampleText: string
      setSampleText: (text: string) => void
      showMotionPath: boolean
      setShowMotionPath: (show: boolean) => void

      // Playhead (fraction 0-1 of phrase lifetime, synced from Player)
      playheadProgress: number
      setPlayheadProgress: (p: number) => void

      // Selection
      selectedProperty: KeyframeableProperty | null
      setSelectedProperty: (p: KeyframeableProperty | null) => void
      selectedKeyframeIndex: number | null
      setSelectedKeyframeIndex: (idx: number | null) => void
    }
    ```

    Implement the store with these key behaviors:
    - `addKeyframe`: Find or create the track for the property. Insert keyframe sorted by time. Add a default `{ type: 'ease-in-out' }` easing for the new segment (maintain easings.length = keyframes.length - 1 invariant).
    - `removeKeyframe`: Remove keyframe at index, remove the corresponding easing segment (remove easing at `min(index, easings.length - 1)`). If only 0-1 keyframes remain after removal, remove the entire track.
    - `updateKeyframeValue` / `updateKeyframeTime`: Update in-place. After time update, re-sort keyframes and re-map easings.

    **2. Create `KeyframePreview.tsx`** — Remotion Player with drag-to-position overlay:

    This component has two layers:
    a) A Remotion `<Player>` that renders the subtitle animation using `SubtitleOverlay`
    b) An absolutely-positioned transparent overlay div on top for drag interaction

    **Remotion Player setup:**
    - Use the `ASPECT_RATIOS` map from research: `{ '16:9': { width: 640, height: 360 }, '9:16': { width: 360, height: 640 }, '1:1': { width: 480, height: 480 } }`
    - Pass `key={aspectRatio}` on the Player to force remount on aspect ratio change
    - Create an inner `BuilderPreviewComposition` component (defined outside KeyframePreview to avoid recreation) that renders:
      - `<AbsoluteFill>` with dark background
      - `<SubtitleOverlay>` with sample phrases built from `sampleText`, using the working preset with current keyframeTracks
    - Build sample phrases: split sampleText into words, create a single CompositionPhrase timed from 0.5s to (durationSec - 0.5s) with the working preset attached
    - Use `playerRef` (useRef<PlayerRef>) for playhead sync — poll with `requestAnimationFrame` to update `playheadProgress` in the store
    - Duration: 5 seconds at 30fps = 150 frames. Phrase runs from 0.5s to 4.0s (frames 15-120).
    - `loop` and `autoPlay` props on Player
    - Memoize `inputProps` with `useMemo` (same pattern as existing AnimationPreview.tsx)

    **Drag overlay:**
    - An absolutely-positioned div covering the Player area, pointer-events auto
    - On `pointerdown` on this div: begin drag, `setPointerCapture`
    - On `pointermove`: compute x% and y% relative to the overlay bounding rect. Call `addKeyframe('x', playheadProgress, xPct)` and `addKeyframe('y', playheadProgress, yPct)` to record/update position at current playhead.
    - On `pointerup`: end drag
    - Cursor: `crosshair` when hovering, `grabbing` when dragging
    - The overlay should be semi-transparent (fully transparent to show the Player below, but catches pointer events)

    **Aspect ratio controls:**
    - Three buttons (16:9, 9:16, 1:1) styled as a button group above the Player
    - Active button highlighted with accent color

    **Sample text input:**
    - A text input above the Player (alongside aspect ratio buttons) for typing custom preview text
    - Default value: "Sample subtitle text here"

    **3. Create `MotionPathOverlay.tsx`:**
    - SVG overlay rendered on top of the Player (same absolute positioning)
    - Read x and y keyframe tracks from the builder store
    - For each pair of consecutive position keyframes, draw a dotted line segment
    - Draw small diamond markers at each keyframe position
    - SVG viewBox matches the current aspect ratio dimensions
    - Only rendered when `showMotionPath` is true in the store
    - pointer-events: none (clicks pass through to the drag overlay)
