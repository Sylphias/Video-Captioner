    For each visible phrase, after computing `cleanPhraseAnimStyles`, also compute keyframe styles:
    ```typescript
    // Compute keyframe-driven position/scale/rotation/opacity
    const phraseProgress = (() => {
      const totalPhraseFrames = Math.round((phraseEnd - phraseStart) * fps)
      const frameIntoPhrase = frame - Math.round(phraseStart * fps)
      return totalPhraseFrames > 0 ? Math.max(0, Math.min(1, frameIntoPhrase / totalPhraseFrames)) : 0
    })()

    const keyframeAnimStyles = computeKeyframeStyles(
      effectivePreset?.keyframeTracks,
      phraseProgress,
      width,
      height,
    )
    ```

    Then merge ALL styles together when building containerStyle:
    ```typescript
    const containerStyle: React.CSSProperties = mergeStyles(
      {
        position: 'absolute',
        top: `${top}%`,
        transform: 'translateY(-50%)',
        left: '5%',
        right: '5%',
        textAlign: 'center',
        fontSize: effectiveStyle.fontSize,
        fontFamily: effectiveStyle.fontFamily,
        fontWeight: effectiveStyle.fontWeight,
        lineHeight: 1.4,
      },
      cleanPhraseAnimStyles,
      keyframeAnimStyles,  // NEW: keyframe position/scale/rotation/opacity
    )
    ```

    **IMPORTANT (from Research Pitfall #1):** `mergeStyles` already handles concatenating transform strings, so keyframe transforms (translateX, translateY, scale, rotate) will be appended to any enter/exit transform strings, not overwrite them. This is correct behavior.

    **IMPORTANT (from Research Open Question #1):** When keyframe x/y tracks exist, they should override the base `top: ${top}%` positioning. To implement this: if the preset has x or y keyframe tracks, omit the base `top` and `left/right` from the base style and let keyframe transforms handle positioning absolutely:

    ```typescript
    const hasPositionKeyframes = effectivePreset?.keyframeTracks?.some(
      t => t.property === 'x' || t.property === 'y'
    ) ?? false

    const baseStyle: React.CSSProperties = hasPositionKeyframes
      ? {
          // When keyframes control position, use absolute center as origin
          position: 'absolute',
          top: '50%',
          left: '50%',
          transform: 'translate(-50%, -50%)',
          textAlign: 'center',
          fontSize: effectiveStyle.fontSize,
          fontFamily: effectiveStyle.fontFamily,
          fontWeight: effectiveStyle.fontWeight,
          lineHeight: 1.4,
        }
      : {
          // Default slot-based positioning
          position: 'absolute',
          top: `${top}%`,
          transform: 'translateY(-50%)',
          left: '5%',
          right: '5%',
          textAlign: 'center',
          fontSize: effectiveStyle.fontSize,
          fontFamily: effectiveStyle.fontFamily,
          fontWeight: effectiveStyle.fontWeight,
          lineHeight: 1.4,
        }

    const containerStyle = mergeStyles(baseStyle, cleanPhraseAnimStyles, keyframeAnimStyles)
    ```

    This way: when position keyframes exist, the text starts at center and keyframe translateX/Y moves it to the authored position. When no position keyframes exist, behavior is identical to before.
