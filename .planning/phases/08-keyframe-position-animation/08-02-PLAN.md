  <action>
    Create `BezierEditor.tsx` — a self-contained SVG-based cubic bezier curve editor (~120-150 lines).

    **Props interface:**
    ```typescript
    interface BezierEditorProps {
      p1x: number  // 0-1
      p1y: number  // unconstrained (overshoot allowed)
      p2x: number  // 0-1
      p2y: number  // unconstrained
      onChange: (p1x: number, p1y: number, p2x: number, p2y: number) => void
      size?: number  // width/height in px, default 160
    }
    ```

    **Implementation details:**
    - SVG viewBox: `"-10 -30 120 160"` — extra space above/below for overshoot handles (p1y/p2y can go beyond 0-1)
    - Draw the curve as `<path d="M 0,100 C {p1x*100},{(1-p1y)*100} {p2x*100},{(1-p2y)*100} 100,0" />` — SVG y-axis is flipped (100=bottom=0.0, 0=top=1.0)
    - Draw a light gray diagonal line from (0,100) to (100,0) as the "linear" reference
    - Draw control handle lines: line from (0,100) to p1, line from (100,0) to p2 — dashed gray
    - Two draggable circles at (p1x*100, (1-p1y)*100) and (p2x*100, (1-p2y)*100) — r=6, filled with accent color
    - Drag interaction: `onPointerDown` on each circle calls `setPointerCapture(e.pointerId)`, sets a `dragging` state ('p1' | 'p2' | null)
    - `onPointerMove` on the SVG element: compute position relative to SVG bounding rect. Convert to normalized 0-1 coords: `x = (clientX - rect.left) / (rect.width) * (viewBox range)`. Map back to 0-1 bezier space. Clamp x to [0, 1], leave y unclamped (allow overshoot).
    - `onPointerUp`: clear dragging state
    - Use `useRef<SVGSVGElement>` for the SVG element to get bounding rect

    **CSS (BezierEditor.css):**
    - `.bezier-editor` container: inline-block, background `var(--surface-2)`, border-radius 6px, padding 8px
    - `.bezier-editor__curve` path: stroke `var(--accent)`, stroke-width 2, fill none
    - `.bezier-editor__handle-line`: stroke `var(--text-muted)`, stroke-width 1, stroke-dasharray 3 3
    - `.bezier-editor__handle`: fill `var(--accent)`, cursor grab; active: cursor grabbing
    - `.bezier-editor__reference`: stroke `var(--text-muted)`, stroke-width 1, opacity 0.3

    Follow the dark theme token pattern from existing components (var(--surface-2), var(--accent), var(--text-muted)).
