Create `undoMiddleware.ts` — a Zustand middleware that captures snapshots for undo/redo.

Implementation approach (snapshot-based, simpler than command pattern):
- Maintain two stacks: `past: StateSnapshot[]` and `future: StateSnapshot[]`
- A `StateSnapshot` captures the undoable parts of the store: `{ session, style, speakerNames, speakerStyles }`
- Before each mutating action, push current state to `past` and clear `future`
- `undo()`: pop from `past`, push current to `future`, restore popped snapshot
- `redo()`: pop from `future`, push current to `past`, restore popped snapshot
- Cap `past` at 50 entries to prevent memory bloat
- Export `useUndoStore` (separate Zustand store) with: `past`, `future`, `undo()`, `redo()`, `canUndo`, `canRedo`, `pushSnapshot(snapshot)`

Create the undo store as a SEPARATE Zustand store (not middleware on subtitleStore) to avoid circular complexity. The pattern:
1. `useUndoStore` holds the stacks and undo/redo actions
2. Mutating actions in `subtitleStore` call `useUndoStore.getState().pushSnapshot(currentState)` before applying changes
3. `undo()` and `redo()` call `useSubtitleStore.setState()` to restore snapshots

Modify `subtitleStore.ts`:
- Import `useUndoStore` from `./undoMiddleware.ts`
- Create a helper `pushUndo()` that captures `{ session, style, speakerNames, speakerStyles }` and calls `useUndoStore.getState().pushSnapshot()`
- Add `pushUndo()` call at the start of these mutating actions: `updateWord`, `splitPhrase`, `mergePhrase`, `addWord`, `addPhrase`, `deleteWord`, `setStyle`, `setSpeakerStyle`, `clearSpeakerStyle`, `renameSpeaker`, `reassignWordSpeaker`
- Do NOT push undo for `setJob`, `resetSession`, `reset` — these are initialization/reset actions, not user edits
- Deep-clone session (structuredClone) for snapshots since it contains nested objects. For `manualSplitWordIndices` (a Set), convert to Array for cloning and back to Set on restore.

Also add a keyboard event listener for undo/redo in SubtitlesPage (will be wired in Task 2).

  </verify>

Create `TextEditor/TextEditor.tsx` — a screenplay-style text editor component.

Props: `seekToTime: (timeSec: number) => void`

**Layout (per user decision — script/screenplay style):**
- Each phrase is a numbered line
- No timestamps visible — clean, focused on words
- Line numbers in a left gutter (dimmed secondary text)
- Phrase text as editable content to the right of the line number
- Speaker badge (if speakers detected) shown next to line number, using existing speaker color system

**Implementation:**
- Read `session.phrases`, `speakerNames` from Zustand store
- Each phrase renders as a `.text-editor__line` row:
  - Left: `.text-editor__line-number` showing `#{phraseIndex + 1}`
  - Optional: `.text-editor__speaker-indicator` (colored dot if speakers detected)
  - Right: `.text-editor__line-content` — a contentEditable div (or a textarea/input) showing all words in the phrase joined by spaces
- **Click-to-seek**: Clicking the line number seeks the preview to `phrase.words[0].start`
- **Inline editing**: Clicking the text content makes it editable. On blur, parse the edited text back into individual words. For text edits per user decision: "Text edits only change the text — original word timestamps stay unchanged. New words get evenly-distributed timestamps within the phrase's time window."
  - Compare new word count vs old word count
  - If same count: update each word's text via `updateWord(globalIdx, { word: newText })`
  - If different count (words added/removed): redistribute timestamps evenly across the phrase's time window (phrase.words[0].start to phrase.words[last].end), then update the session. Create a store action `updatePhraseText(phraseIndex: number, newWords: string[])` in subtitleStore that handles this: keeps the phrase's time window, redistributes word timestamps evenly, updates the session.
- **Enter key (split)**: When user presses Enter inside a line, split the phrase at the cursor position. Determine which word boundary the cursor is at by counting words before cursor position. Call `splitPhrase(phraseIndex, splitBeforeWordIndex)`.
- **Backspace at line start (merge)**: When user presses Backspace and the cursor is at position 0 of the line content, merge this phrase with the previous one. Call `mergePhrase(phraseIndex - 1)` (merges phraseIndex-1 with phraseIndex).
- **Add line**: A "+ Add line" button at the bottom calls `addPhrase(session.phrases.length - 1)`.
- **Delete line**: A small delete icon on hover (similar to WordCell's delete pattern) that deletes all words in the phrase. Or: selecting all text and pressing Backspace/Delete.

**CSS (TextEditor.css) — dark theme, screenplay aesthetic:**
- `.text-editor`: flex column, gap between lines
- `.text-editor__line`: flex row, align-items: start, padding
- `.text-editor__line-number`: fixed width gutter (~40px), `--color-text-secondary`, font-size `--font-size-sm`, cursor pointer (click to seek), monospace or tabular-nums
- `.text-editor__line-content`: flex 1, editable text area, `--color-text-primary`, font-size `--font-size-base`, white-space pre-wrap, min-height 1.4em, padding, border-bottom 1px solid transparent
- `.text-editor__line-content:focus`: border-bottom-color `--color-accent-green`
- `.text-editor__line--active` (optional): subtle highlight for the currently-playing phrase
- Speaker indicator: small colored circle matching speaker lane colors

**Wire into SubtitlesPage:**

  - Only when the page is in "transcribed" state (not during upload/transcribe)
- Add small undo/redo buttons in the top-controls bar (near "Go to subtitle") showing current undo/redo availability
