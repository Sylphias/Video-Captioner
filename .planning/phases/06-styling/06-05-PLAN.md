**Backend waveform extraction (waveform.ts service):**

Create `extractWaveform(videoPath: string, durationSec: number): Promise<Float32Array>` that uses FFmpeg to extract audio waveform amplitude data:
- Run FFmpeg to extract audio as raw PCM: `ffmpeg -i {videoPath} -ac 1 -ar 8000 -f f32le -acodec pcm_f32le pipe:1`
  - `-ac 1`: mono
  - `-ar 8000`: 8kHz sample rate (sufficient for visual waveform — produces ~8000 samples/sec)
  - `-f f32le`: raw 32-bit float little-endian output
  - `pipe:1`: output to stdout
- Collect stdout buffer, convert to Float32Array
- Downsample to ~2000 points total (for the entire video duration) by taking max amplitude in each bucket:
  - `bucketSize = samples.length / 2000`
  - For each bucket, take `Math.max(...absValues)` to get peak amplitude
- Return the downsampled array as Float32Array of 2000 amplitude values (0.0-1.0 normalized)
- Use the same FFmpeg path pattern as existing services (check how normalizeVideo.ts references FFmpeg — likely `/opt/homebrew/bin/ffmpeg`)
- Cache the result in memory (Map keyed by jobId) since extraction only needs to happen once per video

**Backend route (waveform.ts route):**

Create `GET /api/jobs/:jobId/waveform` endpoint:
- Look up job from store, verify it exists and has a normalized video path
- Call `extractWaveform(videoPath, duration)` from the service
- Return JSON: `{ samples: number[], duration: number, sampleRate: number }` where samples is the 2000-point amplitude array
- Register the route in server.ts alongside existing routes

**Frontend hook (useWaveform.ts):**

Create `useWaveform(jobId: string | null)` hook:
- `useState<{ samples: number[]; duration: number } | null>(null)`
- `useEffect` that fetches `/api/jobs/${jobId}/waveform` when jobId is set
- Returns `{ waveform, loading }` — waveform is null until loaded
- Cache in a module-level Map so re-renders don't re-fetch

**WaveformCanvas component:**

Create `WaveformCanvas.tsx`:
- Props: `samples: number[]`, `duration: number`, `scrollLeft: number`, `pixelsPerSecond: number`, `height: number`
- Renders a `<canvas>` element
- Uses `useEffect` + `useRef<HTMLCanvasElement>` to draw the waveform
- Drawing: for each sample, draw a vertical line from center outward (mirrored), using `--color-accent-green` at 30% opacity
- The canvas width = `duration * pixelsPerSecond` (can be wider than the viewport — parent scrolls)
- Responds to `scrollLeft` and `pixelsPerSecond` changes by redrawing
- Use `requestAnimationFrame` for smooth rendering
- Dark background (transparent — the parent provides the background)

**TimingEditor.tsx — the main timing editor component:**

Props: `seekToTime: (timeSec: number) => void`, `jobId: string`

**Layout — horizontal scrollable timeline:**
- A horizontally scrollable container showing the full audio duration
- `pixelsPerSecond` = 100 (so 1 second of audio = 100px; a 60s video = 6000px wide)
- User can scroll horizontally to navigate through the timeline
- Time ruler at the top showing seconds/minutes markers
- WaveformCanvas rendered as background behind phrase blocks

**Phrase blocks:**
- Each phrase renders as a colored block positioned at `left: phrase.words[0].start * pixelsPerSecond`, `width: (lastWord.end - firstWord.start) * pixelsPerSecond`
- Block color: use speaker color if speakers detected (from `--speaker-color-N` CSS vars), else `--color-accent-green` at 40% opacity
- Block height: ~40px
- Inside each block, show the phrase text (truncated if too long)
- Clicking a block selects it (highlight border) and shows the detail panel

**Stacked lanes for overlapping phrases (per user decision):**
- Detect overlapping phrases: if phrase B starts before phrase A ends, they overlap
- Assign overlapping phrases to different vertical lanes (lane 0, lane 1, etc.)
- Lane assignment: simple greedy algorithm — assign to the lowest available lane where there's no temporal overlap
- Each lane has its own vertical row, so overlapping phrases appear stacked
- Typically 1-2 lanes; cap at 3 to prevent excessive height

**Selected phrase detail panel:**
- When a phrase block is clicked, show a detail panel below the timeline (or inline below the selected phrase)
- Detail panel contains:
  - Word-level timestamp editing: for each word in the selected phrase, show the word text + numeric start/end inputs (reuse the scrubby-drag pattern from WordCell)
  - Per-phrase linger slider: range input 0-5s, step 0.1, shows current value. Store linger as `lingerDuration` on the phrase level. Per user decision: "Linger duration is per-phrase, set in the timing stage."
  - Split controls: a split button between each word (same "|" pattern as PhraseRow)
  - Merge button: merge with next phrase (if not last)
