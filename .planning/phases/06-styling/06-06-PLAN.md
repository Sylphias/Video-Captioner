**Stage transition confirmation (Text -> Timing):**

Per user decision: "Switching from Text to Timing: show a brief confirmation ('Text changes saved. X words modified.') then switch view."

Implementation:
- In SubtitlesPage, replace direct `setActiveStage` with a `handleStageChange(newStage: StageId)` function
- When transitioning from 'text' to any other stage:
  - Count modified phrases using a phrase-level comparison: iterate over `session.phrases`, for each phrase join its words into a string (`phrase.words.map(w => w.word).join(' ')`), compare against the original transcript's corresponding phrase text. Count the number of phrases where the joined text differs. This avoids issues when `updatePhraseText` has changed word counts within phrases (word-level index comparison would break down since indices shift after splits, merges, or word count changes within phrases).
  - To get the original text for comparison, store the original phrase texts at transcription time. Add a `useRef<string[]>` in SubtitlesPage that captures `session.phrases.map(p => p.words.map(w => w.word).join(' '))` once when the session is first loaded (when transitioning from transcribing to transcribed state). Update this ref whenever the user re-transcribes.
  - Show a toast/notification: "Text changes saved. {N} phrases modified." (or "No text changes." if N=0)
  - Use a simple inline notification element (not a toast library) — a `.subtitles-page__stage-toast` div that appears for 2 seconds then fades out (CSS animation). Position it near the top of the editor section, over the stage tab bar.
  - Then switch to the new stage

**Timestamp preservation on back-navigation:**

Per user decision: "Going back from later stages to Text: timing adjustments on unchanged words are preserved. Only newly added/modified words get default timestamps."

This is already the default behavior of the store — text edits via `updateWord(idx, { word: newText })` do NOT touch timestamps (they only update the `word` field). The `updatePhraseText` action from Plan 04 only redistributes timestamps for words whose count changed within a phrase. So going back to Text and editing text does not clobber timing.

Add a comment in SubtitlesPage documenting this behavior for clarity.

**Suggested next stage indicator:**

Update StageTabBar to accept an optional `suggestedNext?: StageId` prop. When a stage transition happens, compute the suggested next stage:
- After Text -> suggest Timing
- After Timing -> suggest Speakers
- After Speakers -> suggest Styling
- Styling -> no suggestion (final stage)

The StageTabBar already has visual indicator styling from Plan 03. Wire the `suggestedNext` prop to highlight the tab.

**CSS additions:**
- `.subtitles-page__stage-toast`: absolute positioned notification bar at top of editor section, green background (`--color-accent-green` at 15% opacity), `--color-text-primary` text, slides in from top, fades out after 2s
- `@keyframes toast-fade-in-out`: opacity 0 -> 1 (0.2s) -> hold -> 0 (0.3s at 2s)

   - If speakers detected: verify overlapping phrases stack on separate lanes

4. **Speakers stage (Stage 3)**:

   - Collapse preview panel — verify editor expands to fill space
