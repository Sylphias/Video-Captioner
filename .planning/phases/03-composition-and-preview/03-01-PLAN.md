Use `import()` type for TranscriptWord to avoid runtime dependency on shared-types in this file. Alternatively, import the type directly — either approach is fine since it's type-only.

**SubtitleComposition.tsx:**
- Import `AbsoluteFill`, `OffthreadVideo` from `'remotion'`
- Props: `SubtitleCompositionProps` from `./types`
- Render two layers inside AbsoluteFill:
  - Layer 1: `<OffthreadVideo src={videoSrc} style={{ width: '100%', height: '100%', objectFit: 'contain' }} />`
  - Layer 2: `<AbsoluteFill><SubtitleOverlay words={words} style={style} /></AbsoluteFill>`
- This component must have ZERO hooks, ZERO state, ZERO side effects. It is purely structural.
- Use `objectFit: 'contain'` so videos of different aspect ratios don't crop.

**SubtitleOverlay.tsx:**
- Import `useCurrentFrame`, `useVideoConfig` from `'remotion'`
- Import `TranscriptWord` from `'@eigen/shared-types'`
- Import `StyleProps` from `'./types'`
- Do NOT import anything from `'react'`. All hooks used here come from `'remotion'`.

Implement two pure functions (NOT hooks, NOT components):

1. `findActiveWordIndex(words: TranscriptWord[], currentTimeSec: number): number`
   - Binary search: find the index of the last word whose `start <= currentTimeSec`.
   - This implements standard karaoke UX: the most recently started word stays highlighted even during intra-phrase gaps (brief silences between words within a phrase). This is intentional — returning -1 during gaps would cause flickering that looks broken.
   - Must handle edge cases: empty array returns -1, time before first word returns -1.

2. `groupIntoPhrases(words: TranscriptWord[]): TranscriptWord[][]`
   - Split consecutive words into groups where the gap between word[i].start and word[i-1].end exceeds `PHRASE_GAP_SEC` (1.5 seconds)
   - Empty input returns empty array
   - Single word returns one phrase with one word

Implement the `SubtitleOverlay` component:
- Props: `{ words: TranscriptWord[], style: StyleProps }`
- `const frame = useCurrentFrame()` and `const { fps } = useVideoConfig()`
- `const currentTimeSec = frame / fps`
- Call `groupIntoPhrases(words)` directly (no memoization needed — Remotion controls render cadence per frame, and groupIntoPhrases is O(n) on a typically short words array, so the cost is negligible)
- Find the active phrase: the phrase where `currentTimeSec >= phrase[0].start` and `currentTimeSec <= phrase[last].end + PHRASE_GAP_SEC` (show phrase slightly after last word ends for readability)
- If no active phrase, return `null`
- Find active word index within the active phrase using `findActiveWordIndex`
- Render a positioned div at bottom 10%, left/right 5% margins, centered text
- Map over active phrase words: each word is a `<span>` with `color: (i === activeWordIndex) ? style.highlightColor : style.baseColor`
- Use `marginRight: '0.25em'` between words
- Add text shadow for legibility: `textShadow: '0 2px 4px rgba(0,0,0,0.8), 0 0 2px rgba(0,0,0,0.9)'`
- Key each span as `${word.start}-${i}` (start time + index for uniqueness)
