    1. **Rewrite `subtitleStore.ts`** (`packages/frontend/src/store/subtitleStore.ts`):
       Import `SessionWord`, `SessionPhrase`, `groupIntoPhrases`, `buildSessionPhrases` from `../lib/grouping.ts`. Import `Transcript`, `TranscriptWord`, `VideoMetadata` from `@eigen/shared-types`. Import `StyleProps` from `@eigen/remotion-composition`.

       Store interface (see research RESEARCH.md for full reference):
       ```
       interface SubtitleStore {
         jobId: string | null
         original: Transcript | null          // immutable Whisper output
         videoMetadata: VideoMetadata | null
         session: {
           words: SessionWord[]               // editable copy
           phrases: SessionPhrase[]            // computed from words + manual splits
           manualSplitWordIndices: Set<number> // global word indices where user forced splits
         } | null
         style: StyleProps

         // Actions
         setJob: (jobId: string, transcript: Transcript, videoMetadata: VideoMetadata) => void
         updateWord: (wordIndex: number, patch: Partial<Pick<SessionWord, 'word' | 'start' | 'end'>>) => void
         splitPhrase: (phraseIndex: number, splitBeforeWordIndex: number) => void
         mergePhrase: (phraseIndex: number) => void
         resetSession: () => void
         setStyle: (partial: Partial<StyleProps>) => void
         reset: () => void
       }
       ```

       Key implementation details:
       - `setJob`: Deep-copy `transcript.words` into `session.words`. Compute `session.phrases` via `buildSessionPhrases(words, new Set())`. Store `original: transcript` as-is (immutable reference).
       - `updateWord(wordIndex, patch)`: Update `session.words[wordIndex]` with spread. **If patch contains `start` or `end`**: validate timestamps (start < end for same word; no overlap with neighbors). Then recompute `session.phrases` via `buildSessionPhrases(updatedWords, state.session.manualSplitWordIndices)`. **If patch only contains `word` (text edit)**: update the word text in-place within the existing `session.phrases` structure (find the phrase containing this word by iterating phrases and counting words) ã do NOT recompute phrase boundaries for text-only edits (prevents clobbering manual splits, per pitfall #2 in research).
       - `splitPhrase(phraseIndex, splitBeforeWordIndex)`: Guard: reject if `splitBeforeWordIndex <= 0 || splitBeforeWordIndex >= phrase.words.length`. Splice target phrase into left/right. Compute the global word index of the split point (sum of words in all preceding phrases + splitBeforeWordIndex). Add to `manualSplitWordIndices` Set.
       - `mergePhrase(phraseIndex)`: Guard: reject if `phraseIndex >= phrases.length - 1`. Merge phrases at `phraseIndex` and `phraseIndex + 1`. Remove the corresponding global word index from `manualSplitWordIndices`.
       - `resetSession`: Rebuild session from `original.words` with empty `manualSplitWordIndices`.
       - `setStyle` and `reset`: Same as current (reset clears everything back to defaults).
       - DEFAULT_STYLE stays the same: `{ highlightColor: '#FFFF00', baseColor: '#FFFFFF', fontSize: 48, fontFamily: 'Arial, sans-serif' }`

       Timestamp validation in `updateWord`:
       - If `'start' in patch`: reject if `patch.start! >= word.end` (start must be < end). If `wordIndex > 0`, reject if `patch.start! < session.words[wordIndex - 1].end` (no overlap with previous word).
       - If `'end' in patch`: reject if `patch.end! <= word.start` (end must be > start). If `wordIndex < session.words.length - 1`, reject if `patch.end! > session.words[wordIndex + 1].start` (no overlap with next word).
       - On invalid: silently return current state (no update applied).
