  <action>
    Build bottom-up: WordCell -> PhraseRow -> TranscriptEditor.

    **1. WordCell.tsx** — Inline-editable word with timestamp display.

    Props:
    ```
    interface WordCellProps {
      word: SessionWord
      wordIndex: number           // global index in session.words
      onUpdateText: (wordIndex: number, newText: string) => void
      onUpdateTimestamp: (wordIndex: number, field: 'start' | 'end', value: number) => void
      onSeek: (timeSec: number) => void
    }
    ```

    Implementation:
    - Render word text as an `<input type="text">` with local state via `useState(word.word)`. Size the input to fit its content: use `size={Math.max(text.length, 1)}` attribute and CSS to remove default input chrome (no border by default, minimal padding, inherit font from parent, transparent background).
    - On blur: trim the text. If empty, revert to `word.word`. Otherwise call `onUpdateText(wordIndex, trimmed)`.
    - On keydown: Enter or Escape triggers blur (via `e.currentTarget.blur()`).
    - On click: call `onSeek(word.start)` to seek the video preview.
    - Below the text input, show start/end timestamps as two small `<input type="number" step="0.01" min="0">` inputs. Display them in a subtle secondary style. On blur, `parseFloat` the value; if `isNaN`, revert. Otherwise call `onUpdateTimestamp(wordIndex, 'start' | 'end', parsedValue)`.
    - Timestamp inputs should be small (width ~70px) and shown in `var(--font-size-sm)` / `var(--color-text-secondary)`.
    - IMPORTANT: Use `key={wordIndex + '-' + word.word}` on WordCell from the parent to reset local state when word prop changes externally (re-transcribe scenario, per research pitfall #1).
    - Do NOT use `useCurrentPlayerFrame` anywhere in this component — it would cause 60fps re-renders (research pitfall #6).

    **2. PhraseRow.tsx** — Renders one phrase with its words and split/merge controls.

    Props:
    ```
    interface PhraseRowProps {
      phrase: SessionPhrase
      phraseIndex: number
      globalWordOffset: number    // sum of words in all preceding phrases
      isLast: boolean             // true if this is the last phrase (no merge button)
      onUpdateText: (wordIndex: number, newText: string) => void
      onUpdateTimestamp: (wordIndex: number, field: 'start' | 'end', value: number) => void
      onSplit: (phraseIndex: number, splitBeforeWordIndex: number) => void
      onMerge: (phraseIndex: number) => void
      onSeek: (timeSec: number) => void
    }
    ```

    Implementation:
    - Render a container div with class `phrase-row`. Give it a left border accent (2px solid `var(--color-border)`, or `var(--color-accent-green)` if `phrase.isManualSplit`).
    - Inside, render a flex-wrap row of WordCell components. Between each pair of adjacent WordCells (but NOT before the first word), render a small "split" button — a thin vertical divider/button that shows a scissors icon or "|" text on hover. The split button calls `onSplit(phraseIndex, localWordIndex)` where localWordIndex is the word's position within this phrase (1-indexed from the second word, so the split button between word 0 and word 1 passes `splitBeforeWordIndex=1`).
    - Below the word row, render a "Merge with next" button (only if `!isLast`). Small, subtle, secondary style. Calls `onMerge(phraseIndex)`.
    - Each WordCell receives `wordIndex={globalWordOffset + localIndex}` so the store gets the correct global index.
    - Show phrase number or index label (`#1`, `#2`, ...) on the left side of the row for orientation.

    **3. TranscriptEditor.tsx** — Top-level component reading from Zustand store.

    Props:
    ```
    interface TranscriptEditorProps {
      seekToTime: (timeSec: number) => void
    }
    ```

    Implementation:
    - Read session from store: `const session = useSubtitleStore((s) => s.session)`.
    - Read store actions: `const { updateWord, splitPhrase, mergePhrase } = useSubtitleStore()` — destructure actions directly (they are stable references in Zustand, no need for useCallback).
    - If `!session`, render nothing or a placeholder.
    - Create handler callbacks:
      - `handleUpdateText = (wordIndex, newText) => updateWord(wordIndex, { word: newText })`
      - `handleUpdateTimestamp = (wordIndex, field, value) => updateWord(wordIndex, { [field]: value })`
    - Render a header showing phrase count: `"{session.phrases.length} phrases, {session.words.length} words"`.
    - Render a scrollable list of `PhraseRow` components, one per phrase. Compute `globalWordOffset` for each phrase by summing preceding phrase word counts.
    - Pass `seekToTime` prop down as `onSeek` to PhraseRow and ultimately to WordCell.

    **4. TranscriptEditor.css** — Styles for all three components.
