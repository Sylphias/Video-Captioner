## Component Inventory

New components required for this phase. All follow the co-located CSS pattern (e.g. `ProjectsPage.tsx` + `ProjectsPage.css`).

### ProjectsPage

Full-page view rendered when the "Projects" tab is active. Contains:
- Page heading "Your Projects" (Heading typography)
- Responsive card grid
- Empty state (when no projects exist)

Layout: `display: grid; grid-template-columns: repeat(auto-fill, minmax(240px, 1fr)); gap: var(--spacing-lg);`
Page padding: `var(--spacing-xl)` all sides.

### ProjectCard

Individual project card. Dimensions: width fluid (grid cell), height auto.

Structure (top to bottom):
1. **Thumbnail area** ‚Äî 160px fixed height, `background: var(--color-bg-base)`, `border-radius: 6px 6px 0 0`. Contains `<img>` (cover fit) or a dark placeholder with a centered film-strip icon (text: "‚ñ∂" or Unicode `‚¨õ` as fallback).
2. **Metadata area** ‚Äî `padding: var(--spacing-sm) var(--spacing-md)`. Contains:
   - Project name: Body (16px/400)
   - Last edited: Label (14px/400), `--color-text-secondary`
   - Duration badge: Label (14px/400), `--color-text-disabled`
3. **Card container** ‚Äî `background: var(--color-bg-elevated)`, `border: 1px solid var(--color-border)`, `border-radius: 6px`, `cursor: pointer`
4. **Hover state** ‚Äî `background: var(--color-bg-hover)`, `border-color: var(--color-border-focus)`, transition: `150ms ease` (`--transition-fast`)

Context menu trigger: right-click on card opens `ProjectContextMenu`. No visible trigger button on card face.

### CreateNewCard

Visual variant of `ProjectCard` used as the last item in the grid.

Structure:
- Full card area (same height as project cards): dashed border `2px dashed var(--color-border)`, `border-radius: 6px`
- Centered content: "+" symbol (20px, weight 600) above label "New Project" (14px, `--color-text-secondary`)
- Hover: dashed border color ‚Üí `--color-accent-green`, text color ‚Üí `--color-text-primary`
- Click: opens upload zone inline (existing `UploadZone` component) or transitions to editing view with upload focus

### ProjectContextMenu

Right-click context menu. Matches the existing `keyframe-track-row__context-menu` pattern (position:fixed at click coordinates).

Structure:
- Container: `background: var(--color-bg-elevated)`, `border: 1px solid var(--color-border)`, `border-radius: 6px`, `min-width: 160px`, `box-shadow: 0 4px 16px rgba(0,0,0,0.4)`
- Items: `padding: var(--spacing-sm) var(--spacing-md)`, `min-height: 32px`, Body (16px/400)
- Items: Rename, Duplicate, Re-transcribe (normal color: `--color-text-primary`)
- Item: Delete (text color: `--color-error`)
- Item hover: `background: var(--color-bg-hover)`
- Separator before Delete: `1px solid var(--color-border)`
- Dismiss: click outside or Escape key

### DeleteConfirmDialog

Modal dialog for project deletion. Matches `find-replace-modal` pattern.

Structure:
- Backdrop: `position: fixed; inset: 0; background: rgba(0,0,0,0.6)`
- Dialog box: `background: var(--color-bg-surface)`, `border: 1px solid var(--color-border)`, `border-radius: 8px`, `width: 400px`
- Header: Heading (18px/600) "Delete Project?", `padding: var(--spacing-md)`, `border-bottom: 1px solid var(--color-border)`
- Body: `padding: var(--spacing-md)`. Copy: see Copywriting Contract below.
- Footer: `padding: var(--spacing-md)`, right-aligned button row, `gap: var(--spacing-sm)`
  - Keep Project button: `background: var(--color-bg-elevated)`, `border: 1px solid var(--color-border)`, `color: var(--color-text-primary)`
  - Delete button: `background: transparent`, `border: 1px solid var(--color-error)`, `color: var(--color-error)`
  - Delete button hover: `background: rgba(224,84,84,0.1)`
- Dismiss: Keep Project button, Escape key, or backdrop click

### RenameInput

Inline project name editing within the card's metadata area.

Trigger: "Rename" selected from context menu.
Behavior: Project name text replaced with a full-width `<input type="text">` pre-filled with current name.
Commit: Enter key or blur.
Cancel: Escape key (reverts to original name).
Style: `background: var(--color-bg-base)`, `border: 1px solid var(--color-accent-green)`, `border-radius: 3px`, `padding: var(--spacing-xs) var(--spacing-xs)`, same font as the replaced text (16px/400).

### AutoSaveIndicator

Subtle status indicator in the top-right area of the editing view (SubtitlesPage header zone or fixed bottom-right corner).

States:
- Saving: Label (14px/400) "Saving‚Ä¶" in `--color-text-secondary`
- Saved: Label (14px/400) "Saved" with a 6px filled circle in `--color-accent-green` before the text
- Error: Label (14px/400) "Save failed ‚Äî retrying‚Ä¶" in `--color-error`
- Idle (no recent save): hidden (opacity: 0)

Transition: fade in on state change, fade to opacity:0 after 3 seconds of "Saved" state.
Position: fixed bottom-right, `bottom: var(--spacing-md); right: var(--spacing-md)`.

### Nav Restructure (TabNav update)

Top nav tabs change from `[Subtitles, Animation Builder]` to `[Projects, Animation Builder]`.

- `TABS` array in `App.tsx` updated: `{ id: 'projects', label: 'Projects' }` replaces `{ id: 'subtitles', label: 'Subtitles' }`
- Active tab: existing `tab-nav__tab--active` class (accent-green underline or color ‚Äî matches whatever current active styling uses)
- No new tab styles needed

## Interaction Contracts

### Project Card ‚Äî Click

Single click on a `ProjectCard` (not on a context menu trigger): navigate to editing view for that project. The "Projects" tab remains visually active in the nav; the `SubtitlesPage` renders in project-scoped mode.

No loading state needed unless the project's auto-save blob is large. If load time >300ms, show the existing page background while the store rehydrates silently.

### Project Card ‚Äî Right-Click

Opens `ProjectContextMenu` at `{ x: event.clientX, y: event.clientY }` using `position: fixed`. Only one context menu can be open at a time; opening a second closes the first.

Keyboard: context menu items navigable with arrow keys; Enter to activate; Escape to close.

### Create New Card ‚Äî Click

Transitions to editing view in "new project" mode ‚Äî same as current app startup state. Upload zone is the primary content. Video filename becomes the initial project name on upload completion (D-10).

### Auto-save

Debounce: 4 seconds after last detected change to `subtitleStore` state (midpoint of 3-5 second range from D-04).
Trigger: `useEffect` watching store state hash or version counter.
No manual save button anywhere in the UI.

### Delete Flow

1. User right-clicks card ‚Üí context menu appears
2. User clicks "Delete" ‚Üí `ProjectContextMenu` closes, `DeleteConfirmDialog` opens
3. User clicks "Delete Project" ‚Üí dialog closes, card animates out (`opacity: 0, height: 0` over 200ms), project removed from grid
4. User clicks "Keep Project" or presses Escape ‚Üí dialog closes, no change

### Rename Flow

1. User right-clicks card ‚Üí context menu ‚Üí "Rename"
2. Card metadata area: project name text replaced with `RenameInput`
3. User edits text, presses Enter or clicks away ‚Üí name saved, input replaced with updated text
4. User presses Escape ‚Üí input replaced with original name, no save
